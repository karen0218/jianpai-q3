# 改版 Demo 线下测试 · 主线任务脚本

本脚本用于 2-3 条 demo 主线的现场测试。每条主线可以独立使用，也可以组合使用。

## 主线 A：首页启动路径

路径：首页 → 情绪打卡 → AI 接住 → 今日推荐

### 测试目标

验证新版首页是否能让学生自然完成“先表达状态，再获得下一步”的体验。

### 场景设定

你今天上学有点累，心里有点烦，但也说不上来具体为什么。你打开这个 App，看看它会怎么接住你。

### 主持话术

1. “你先看这个首页，别急着点。你觉得它想让你先做什么？”
2. “如果你今天有点烦，你会从哪里开始？”
3. “现在试着记录一下今天的状态。”
4. “做完后看一看，它有没有让你觉得它知道你现在是什么状态。”
5. “它推荐了下一步，你会愿意点吗？为什么？”

### 关键观察点

| 观察点 | 记录方式 |
| --- | --- |
| 首页第一行动是否明确 | 明确 / 模糊 / 不明确 |
| 情绪打卡是否自然 | 自然 / 勉强 / 不自然 |
| AI 接住是否被感知 | 有 / 一般 / 没有 |
| 推荐内容是否有理由 | 有理由 / 模糊 / 像任务列表 |
| 是否愿意继续点推荐 | 愿意 / 犹豫 / 不愿意 |

### 结束问题

1. 这个首页有没有告诉你“现在可以先做什么”？
2. 它说的话像不像在回应你刚才选的状态？
3. 你会愿意继续做它推荐的那件事吗？

## 主线 B：AI 伙伴路径

路径：AI 对话 → 三 IP 理解/选择 → 继续聊 → 情绪记忆

### 测试目标

验证三 IP 是否被理解，以及“它记得我”是否提升信任感。

### 场景设定

你最近有一个烦恼：学习有点累，偶尔也不知道该跟谁说。你想找一个 AI 伙伴聊聊。

### 主持话术

1. “你看这三个伙伴，你会先找谁？为什么？”
2. “如果你只是想有人陪你说说话，会选谁？”
3. “如果你很难过，会选谁？”
4. “如果你在想选科、未来方向，会选谁？”
5. “现在选一个，和它聊 2-3 轮。”
6. “如果它提到你上次说过的事，你有什么感觉？”

### 关键观察点

| 观察点 | 记录方式 |
| --- | --- |
| 是否能说出三 IP 差异 | 能 / 模糊 / 不能 |
| 选择是否快速 | ≤30 秒 / 30-60 秒 / >60 秒 |
| 是否愿意真实表达 | 愿意 / 半真实 / 不愿意 |
| 回应是否像对应 IP | 像 / 一般 / 不像 |
| 记忆设计是否被注意 | 主动注意 / 被提醒后注意 / 没注意 |
| 记忆是否带来信任 | 增强 / 无明显变化 / 反感 |

### 结束问题

1. 你觉得这三个伙伴有什么不同？
2. 它记得你之前的状态，你觉得安心还是奇怪？
3. 你希望它记住哪些事？哪些事不希望它记住？

## 主线 C：完成反馈路径

路径：脑之岛/测评/放松训练 → 完成反馈 → 成长记录 → AI 回应

### 测试目标

验证“做完之后”是否形成价值感，避免模块孤岛。

### 场景设定

你刚刚完成了一个小训练或测评。你想知道自己做得怎么样，以及这件事对你有什么帮助。

### 主持话术

1. “现在你完成了这个训练/测评。你觉得接下来应该看到什么？”
2. “看看系统给你的反馈，你能看懂自己刚才完成了什么吗？”
3. “你觉得这个记录对你有用吗？”
4. “如果 AI 伙伴对你说一句话，你希望它说什么？”
5. “看到这个成长记录后，你会想明天再来吗？”

### 关键观察点

| 观察点 | 记录方式 |
| --- | --- |
| 完成结果是否明确 | 明确 / 模糊 / 不明确 |
| 奖励或成长记录是否被感知 | 有 / 一般 / 没有 |
| AI 回应是否自然 | 自然 / 一般 / 生硬 |
| 是否理解这件事和情绪成长有关 | 理解 / 模糊 / 不理解 |
| 是否增强下次使用意愿 | 增强 / 无变化 / 降低 |

### 结束问题

1. 你觉得刚才做完之后，系统有没有“回应你”？
2. 这个记录你之后会想看吗？
3. 如果正式上线，你觉得这个功能还缺什么？

## 主线优先级建议

如果只能测 2 条：

1. 优先测主线 A：首页启动路径。
2. 优先测主线 B：AI 伙伴路径。

如果资源允许再测主线 C，因为它更偏留存和长期价值，需要更完整的 demo 才能测准。
