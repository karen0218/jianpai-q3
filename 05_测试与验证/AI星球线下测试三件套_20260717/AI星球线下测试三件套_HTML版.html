AI星球线下测试三件套

这套材料用于把 9 月学生线下测试拆成三件清楚的事：先用当前 APK 找准体验病灶，再用 7-8 月优化 Demo 验证方案，最后用统一指标判断哪些主线进入正式迭代。

不修改原 0706 工具包 适配 AI星球 APK v1.0.2801 支持 2-3 条 Demo 主线 可打印 / 可评审

建立现状基线，确认学生真实触发哪些卡点，输出 Demo 设计输入。

APK 执行脚本

开场话术：今天不是来考你，也不是看你会不会用 App。我们想知道这个 App 对学生来说好不好懂、哪里容易卡住。如果你不知道点哪里，可以直接说出来，这是产品需要改进的地方。

故事 1：首页到 AI 对话

场景：你今天心情不太好，第一次打开这个 App，想看看它能不能帮你。

  1. 你先看看首页，告诉我你注意到了什么。
  2. 如果今天心情不太好，你会先点哪里？
  3. 你会找哪个 AI 伙伴聊？为什么？
  4. 现在试着和它聊 3 轮，可以说一个不太严重但真实的烦恼。

故事 2：情绪打卡和反馈

场景：你想记录一下今天的心情，看看系统会不会理解你。

  1. 请你试着记录今天的心情。
  2. 做完后告诉我，你觉得它有没有表达出你现在的状态。
  3. 如果系统接下来推荐你做一件事，你期待是什么？

故事 3：脑之岛训练与跨模块回流

场景：你想训练一下注意力，老师说脑之岛里有练习，你来试试看。

  1. 你想找脑之岛做注意力训练，会从哪里进入？
  2. 你觉得这几个岛分别是干什么的？
  3. 选一个你觉得适合注意力训练的内容试试。
  4. 完成后你觉得自己得到了什么反馈？
  5. 如果你现在想和 AI 伙伴说刚才做完了训练，会怎么做？

危机响应观察

此环节不让学生扮演危机用户。由研究员使用测试账号输入预设语句，学生只观察系统响应。
  • 你看懂它现在想让你做什么吗？
  • 这段提示让你觉得安心、紧张，还是困惑？
  • 如果这是你同学遇到的情况，你觉得这个提示有帮助吗？

02 改版 Demo 线下测试工具包

Demo 测试不是复测所有功能，而是验证 2-3 条改版主线是否成立。它的核心指标是理解、开始、完成、被理解感和继续使用意愿。

主线 A 首页启动路径

首页 → 情绪打卡 → AI 接住 → 今日推荐。验证首屏是否更清楚。

主线 B AI 伙伴路径

AI 对话 → 三 IP 理解/选择 → 继续聊 → 情绪记忆。验证陪伴感是否成立。

主线 C 完成反馈路径

脑之岛/测评/放松 → 完成反馈 → 成长记录 → AI 回应。验证做完后的价值感。

Demo 结果分类

类型含义后续动作
明显改善更快理解、更少提示、更愿意继续进入开发候选
方向正确但表达不清认可价值，但入口或文案不清继续优化设计
学生无感设计团队认为重要，但学生没感知降低优先级或换表达
引入新困惑解决旧问题但带来新成本重新设计
原型限制导致误判因 demo 不完整导致卡住需二次验证

Demo 主线脚本

主线 A：首页启动路径

场景：你今天上学有点累，心里有点烦，但也说不上来具体为什么。你打开这个 App，看看它会怎么接住你。

  1. 你先看这个首页，别急着点。你觉得它想让你先做什么？
  2. 如果你今天有点烦，你会从哪里开始？
  3. 现在试着记录一下今天的状态。
  4. 做完后看一看，它有没有让你觉得它知道你现在是什么状态。
  5. 它推荐了下一步，你会愿意点吗？为什么？

主线 B：AI 伙伴路径

场景：你最近有一个烦恼：学习有点累，偶尔也不知道该跟谁说。你想找一个 AI 伙伴聊聊。

  1. 你看这三个伙伴，你会先找谁？为什么？
  2. 如果你只是想有人陪你说说话，会选谁？
  3. 如果你很难过，会选谁？
  4. 如果你在想选科、未来方向，会选谁？
  5. 现在选一个，和它聊 2-3 轮。
  6. 如果它提到你上次说过的事，你有什么感觉？

主线 C：完成反馈路径

场景：你刚刚完成了一个小训练或测评。你想知道自己做得怎么样，以及这件事对你有什么帮助。

  1. 现在你完成了这个训练/测评。你觉得接下来应该看到什么？
  2. 看看系统给你的反馈，你能看懂自己刚才完成了什么吗？
  3. 你觉得这个记录对你有用吗？
  4. 如果 AI 伙伴对你说一句话，你希望它说什么？
  5. 看到这个成长记录后，你会想明天再来吗？

改善判定

强改善

完成率提升、提示率下降、卡点减少、满意度/被理解感/继续意愿提升，且无明显新增困惑。

中等改善

满足 1-2 条改善，但仍有明显困惑。方向保留，继续打磨入口、文案或层级。

负向改善

解决一个旧问题，但引入更大的新问题。不进入开发，回到设计阶段。

一句话结论

示例：本轮测试显示，“首页先接住状态，再由 AI 推荐下一步”的方向显著降低首屏决策成本，建议作为 9 月后正式迭代的第一优先级。

下一步建议

  1. 先用 APK 优化包测当前版本，输出真实卡点和优先主线。
  2. 7-8 月只做 2-3 条 Demo 主线，不做大而全改版。
  3. 资源紧时优先做“首页启动路径”和“AI 伙伴路径”。
  4. Demo 测试不做完整 SUS，重点看理解、开始、被理解感和继续使用意愿。
  5. 对比报告只让证据最强的主线进入正式开发。
