# AI星球线下测试三件套 · 使用总览

版本：2026-07-17  
适用对象：产品、UX、用研、研发负责人  
目标：把 9 月线下学生测试拆成“现状诊断 → Demo 验证 → 对比决策”三件事，避免一次测试同时承担太多目标。

## 为什么拆成三套

原 `减压星球_线下易用性评估套装0706` 主要服务当前 APK 的易用性诊断，适合回答“现在哪里不好用”。但 7、8 月还会产出 2-3 条优化 demo，后续还要判断 demo 是否值得进入正式迭代，因此需要三套材料各自承担不同任务。

1. APK 当前版本易用性评估优化包  
   目标：建立当前 APK 的现状基线，确认 P1-P11 哪些是真用户会遇到的问题。

2. 改版 Demo 线下测试工具包  
   目标：验证 2-3 条优化主线是否真的改善理解、开始、完成、被理解感和继续使用意愿。

3. APK vs Demo 对比分析工具包  
   目标：把两轮测试数据转成产品决策，判断哪些优化进开发、哪些继续打磨、哪些暂缓。

## 推荐时间线

7 月中下旬：用优化后的 APK 测试包做当前版本基线测试。  
7-8 月：根据 APK 测试结果制作 2-3 条优化 demo。  
8 月底或 9 月：用 Demo 测试包做第二轮线下测试。  
第二轮测试后 3-5 天：用对比分析工具包产出迭代决策报告。

## 三套工具包目录

### 01_APK当前版本易用性评估优化包

- `A_APK测试包优化说明.md`：对 0706 工具包的优化方向和修正清单。
- `B_APK执行手册_精简版.md`：适合现场主持使用的新版流程。
- `C_APK快速记录表.md`：降低现场记录负担，一人一份。

### 02_改版Demo线下测试工具包

- `A_Demo测试执行手册.md`：Demo 测试的目标、样本、流程和主持原则。
- `B_Demo主线任务脚本.md`：2-3 条主线的任务脚本。
- `C_Demo记录表.md`：Demo 专用记录表，强调感知改善和新困惑。

### 03_APK_vs_Demo对比分析工具包

- `A_对比分析框架.md`：两轮数据如何对齐、怎么判断有效改善。
- `B_对比报告模板.md`：最终产品决策报告模板。

## 核心原则

不要让同一场测试同时回答所有问题。  
第一轮测 APK，是找准病灶；第二轮测 Demo，是验证药方；第三步做对比，是决定哪些方案进入正式迭代。

## 建议保留的统一指标

为保证两轮测试能对比，APK 和 Demo 都建议保留以下指标：

| 指标 | 记录方式 | 用途 |
| --- | --- | --- |
| 任务完成状态 | 成功 / 需提示 / 失败 | 判断流程是否可独立完成 |
| 首次停顿点 | 记录触点和原话 | 找到理解障碍 |
| 关键路径耗时 | 秒或分钟 | 判断效率是否提升 |
| 情绪标记 | 顺利 / 一般 / 困难 | 捕捉体验峰谷 |
| 即时满意度 | 1-5 分 | 每条主线的主观顺畅度 |
| 被理解感 | 1-5 分 | AI 陪伴类体验的核心指标 |
| 继续使用意愿 | 1-5 分 | 判断是否有留存潜力 |
| 用户原话 | 尽量逐字记录 | 支撑产品决策和汇报 |
