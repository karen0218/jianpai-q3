AI星球 / 减压星球 · Day 03-05

竞品分析成果 · Codex视角

这份成果基于公开资料、应用商店信息和产品体验机制分析。它不是“竞品功能罗列”，而是服务 20 天 demo 的设计判断：减压星球应该学什么、不学什么，以及如何转成 Flow A/B/C 的可测试原型。

一句话结论 减压星球不应该照搬任何一个竞品。最优路径是：学 Finch 的陪伴式自我照顾、How We Feel 的低压力情绪表达、Headspace/Ebb 与 Wysa 的 AI边界和下一步建议、Elevate/Lumosity 的训练结果解释、Duolingo 的轻量即时反馈，但要去掉强竞争、强打卡焦虑和成人心理产品的冷感。

Flow A

2. Flow A：首页启动路径竞品分析

目标路径：首页 → 今日状态 → 情绪打卡 → AI接住 → 今日小行动。

竞品 降低用户压力 引导下一步 反馈方式 我的转译判断
Finch 它把“我要自律/我要疗愈”的压力，换成“我照顾自己，伙伴也会成长”的轻负担关系。 从 check-in 进入小目标、自我照顾练习和奖励，不让用户面对功能海。 完成后用伙伴成长、奖励、鼓励形成“我做到了”的即时感。 减压星球首页要从“功能入口页”变成“今日状态起点”。AI伙伴不是另一个功能，而是打卡后的承接者。
How We Feel 它把情绪表达拆成颜色、词语、程度和标签，降低“不知道怎么说”的压力。 打卡后连接到模式洞察和调节策略，让记录不是孤立数据。 反馈偏理解型：帮助用户找到词、看到模式，而不是评价对错。 减压星球需要“说不清也可以”的选项。小学生可以更直接，高中生可以更克制，但都不要逼用户长篇解释。
Headspace / Ebb AI先承接表达，减少用户自己判断“我该做哪个练习”的负担。 根据聊天内容推荐 app 内练习和活动。 反馈由“我理解你 + 这个练习适合你”的推荐理由组成。 今日小行动必须带推荐理由，只给一个主推荐。多推荐会把压力还给用户。
Flow A 设计结论 首屏要先解决“我现在怎么开始”，再解决“产品有什么功能”。最适合的结构是：状态选择作为第一行动，AI回应作为情绪承接，今日小行动作为低成本下一步。
Flow B

3. Flow B：AI伙伴路径竞品分析

目标路径：全局AI入口 → 三IP选择/切换 → 对话回应 → 记忆/边界 → 下一步建议。

竞品/标杆 降低用户压力 引导下一步 反馈方式 我的转译判断
Wysa 用 AI 聊天和结构化问题降低“直接找真人咨询”的门槛。 从表达进入自助工具、情绪练习、外部支持或其他资源。 反馈强调安全、支持、自助，不做诊断承诺。 三IP要变成不同支持路径：小星陪伴、吉塔稳定、罗素拆解。不能只换头像和口头禅。
Headspace / Ebb 允许语音或文字表达，并强调 AI 记住重要内容，降低重复解释的疲惫。 聊天后推荐个性化活动，把情绪表达导向实际练习。 反馈包含“理解、记忆、推荐、安全设计”。 AI星球可以做轻量记忆，但必须给控制权：记住/不记/删除。记忆越具体，越要谨慎。
988 / Crisis Text Line / Trevor 通过“接下来会发生什么”降低求助不确定性；等待期间也提供安抚动作。 风险上升时，路径转向真人支持、紧急服务或可信帮助。 反馈先确认安全，再提供支持和资源；保密边界说清楚。 减压星球不能写“绝对保密”。更好的说法是“大多数分享会被尊重，但明确危险时优先保护安全”。
Flow B 设计结论 AI伙伴的价值不是“更会聊天”，而是“知道该找谁、怎么说、说完之后有下一步”。安全边界不应该像法律免责声明，而应该像产品体验的一部分。
Flow C

4. Flow C：完成反馈路径竞品分析

目标路径：脑之岛/测评/放松训练 → 结果解释 → 奖励/成长记录 → AI回应。

竞品 降低用户压力 引导下一步 反馈方式 我的转译判断
Elevate 每天给个性化训练计划，减少“我该练什么”的选择压力。 通过训练日历、连续练习、适应难度引导持续使用。 反馈连接到能力维度和表现追踪。 脑之岛完成后不要只显示勋章，要说“今天练到了什么能力”，并把结果带回首页/AI。
Lumosity 把认知训练做成短游戏，降低训练的严肃感。 通过能力分类、LPI/洞察、进度报告引导继续练。 用图表和报告解释进步，但也可能带来比较压力。 减压星球要学“能力解释”，不要学“分数压迫”。结果语言应是“今天练习了专注”，不是“你专注力较差”。
Duolingo 任务极短，反馈即时，失败成本低。 奖励、streak、下一课让用户自然继续。 强即时反馈、经验、成就和连续使用提示。 可以学即时反馈和短任务，不建议学排行榜和强 streak 压力。减压产品不应把用户拖进焦虑式打卡。
Finch 把完成任务变成陪伴关系里的温柔回应，不像被考核。 完成后回到伙伴成长、奖励和下一项小目标。 反馈有人情味，能让用户感觉“我刚才做的事被看见”。 完成反馈要由 IP 接住：完成训练后，伙伴说一句看见你的话，而不是只弹奖励。
Flow C 设计结论 完成反馈要回答三件事：我刚才做了什么、这对我有什么帮助、它有没有被记录。奖励可以有，但不能成为唯一反馈。

5. 我提炼出的 6 条设计策略

01

先状态，后功能

学生不是来逛功能的，而是带着某种情绪状态进入。首页第一屏应该先问“今天怎么样”。

02

允许说不清

情绪产品最该保护的是表达门槛。“说不清”不是异常状态，而是青少年真实状态。

03

AI要负责收束

AI不能只陪聊，它要把表达收束到小行动、记录、练习或安全支持。

04

角色差异要工作流化

三IP不是设定资产，而是三种支持方式：陪伴、稳定、分析。

05

完成后要有回声

训练、测评、写信做完后必须被首页、AI或成长记录接住，否则仍然是模块孤岛。

06

安全边界要可信

不要用“高度保密不会泄漏”这种绝对承诺。对未成年人产品，可信比好听更重要。

6. 明确不要照搬的地方

来源 不建议照搬 原因 更适合减压星球的做法
Finch 过强宠物养成和装扮经济。 高中生可能觉得低幼，且会把核心价值带偏。 保留“伙伴看见我”和轻奖励，不做重养成。
How We Feel 过细的情绪词矩阵。 对首次使用和低年级学生可能认知负担高。 先做 6-12 个状态词，再逐步展开颗粒度。
Wysa / Headspace 成人心理健康产品的冷静合规表达。 学生可能读不懂，或觉得这不是“伙伴”。 把合规写成温和、短句、场景化边界。
Duolingo 强 streak、排行榜、损失厌恶。 减压产品不能制造新的打卡焦虑。 使用“回来看看也很好”的低压力连续感。
Elevate / Lumosity 过强分数、排名、能力比较。 学生可能把认知训练理解成考试和评价。 结果解释改为“练到了什么”，不评价“好坏”。

7. 对 20 天 Demo 的直接落地建议

Demo模块 必须落地的竞品启发 页面/交互要求 测试时要验证
首页 Finch 的轻起点 + How We Feel 的状态表达。 首页主行动固定为“今天感觉怎么样”；功能入口弱化到第二层。 学生 3 秒内是否知道第一步。
情绪打卡 How We Feel 的情绪词辅助。 状态词短、可跳过、可“说不清”；不做长问卷。 学生是否觉得表达容易。
AI接住 Headspace/Ebb 的理解后推荐。 AI先回应状态，再给一个有理由的小行动。 学生是否觉得“它知道我刚才选了什么”。
三IP对话 Wysa 的结构化支持 + Headspace 的记忆感。 三IP按“陪伴/稳定/分析”分工；记忆必须可控。 学生是否能说出三IP差异。
安全边界 988/CTL/Trevor 的“会发生什么”和保密例外说明。 短句说明 AI 不替代真人帮助，危险时优先安全支持。 学生是否觉得可信、不被吓退。
完成反馈 Elevate/Lumosity 的能力解释 + Finch 的伙伴反馈。 完成后显示“练到了什么”、奖励原因、成长记录、AI一句回应。 学生是否知道做这件事有什么用。
最终优先级 Flow A 和 Flow B 的高保真优先级高于 Flow C。Flow C 不要做全脑之岛，只保留“完成页 + 成长记录 + AI回应 + 首页回流”的轻闭环即可。
