Day 01-02 · 核心Flow定范围与验证假设

本文件用于把 7-8 月优化 demo 的范围定住：20 天内只做能和当前 APK 形成对照、能被学生线下测试验证的 2-3 条主线，不做完整新版本。

最终范围结论

建议默认做 2.5 条主线。 Flow A 和 Flow B 必做且高保真；Flow C 做成可测试闭环，如果时间被压缩，可以降级为中保真或只做关键结果页。
必做

Flow A：首页启动路径

首页 → 今日状态 → 情绪打卡 → AI接住 → 今日小行动。

验证学生打开后能不能快速知道第一步做什么。

必做

Flow B：AI伙伴路径

AI入口 → 三IP选择/切换 → 对话回应 → 记忆/边界 → 下一步建议。

验证产品最核心的差异化资产能不能建立信任。

条件做

Flow C：完成反馈路径

脑之岛/测评/放松训练 → 结果解释 → 成长记录 → AI回应。

验证做完后是否有价值感和继续使用理由。

3条主线定义

P0主线

Flow A：首页启动路径

打开首页 → 选择今日状态 → 情绪打卡 → AI接住 → 推荐今日小行动 → 完成后反馈

为什么做

当前 APK 的首屏有情绪打卡和 AI 伙伴两个强入口，但没有明确视觉主角。学生第一次打开时容易停顿，不知道应该先打卡、先聊天，还是去看别的模块。

验证假设

如果首页先询问/呈现“今天的状态”，再由 AI 伙伴接住并给一个小行动，学生的首屏停顿会减少，情绪打卡会更自然，推荐内容会更像“为我而来”。

目标用户小学高年级、初中、高中学生；尤其是“有点烦但说不清”的轻压力用户。
核心体验词容易开始、低压力表达、被接住、知道下一步。
必须出现的页面/状态新版首页、今日状态选择、情绪打卡结果、AI回应卡、今日小行动推荐、完成反馈。
不要做成不要把首页做成一堆功能入口；不要让推荐像任务列表；不要强迫用户长篇表达。
9月测试问题学生是否能在 3 秒内说出先做什么？是否愿意点今日小行动？是否觉得 AI 回应了刚才的状态？
P0主线

Flow B：AI伙伴路径

全局AI入口 → 三IP选择/切换 → 输入烦恼 → 伙伴回应 → 轻量记忆 → 安全边界 → 下一步建议

为什么做

AI伙伴是产品最强差异化，但当前体验里角色分工、持续陪伴、记忆感和合规提示都没有充分形成一个可信关系。这个 Flow 是决定产品是不是“成长伙伴”的关键。

验证假设

如果三 IP 的分工能被快速理解，并在对话里体现不同回应方式，同时用温和方式表达记忆和安全边界，学生会更愿意表达，也更能相信这个 AI 不是普通问答工具。

目标用户愿意尝试 AI 对话的学生，以及对心理咨询/倾诉有犹豫的学生。
核心体验词可信、像伙伴、角色清楚、不过界、有下一步。
必须出现的页面/状态全局AI入口、三IP选择页/切换控件、对话页、记忆提示、免责声明改写、安全边界提示、行动建议。
不要做成不要只换头像和话术；不要把免责声明放成持续打断；不要承诺“所有内容都不会告诉任何人”。
9月测试问题学生是否能说出小星/吉塔/罗素分别适合什么？是否愿意继续聊？是否觉得记忆安心还是冒犯？
P1主线

Flow C：完成反馈路径

脑之岛/测评/放松训练 → 完成结果 → 低压力解释 → 奖励/成长记录 → AI回应 → 明天再来

为什么做

当前脑之岛、解忧信箱、星星商店和成长奖励更像各自独立的小模块。学生做完一件事后，如果没有被回应、没有记录、没有外溢到首页和 AI，就很难形成持续使用理由。

验证假设

如果完成训练/测评后，系统能用低压力语言解释“你刚刚完成了什么”，并把结果同步到成长记录和 AI 回应，学生会更理解它和情绪成长的关系，也更愿意下次继续。

目标用户完成过训练、测评、放松任务或写信的学生。
核心体验词做完有回应、看得懂、有进步感、愿意回来。
必须出现的页面/状态训练/测评完成页、结果解释、奖励获得、成长记录条目、首页回流状态、AI一句回应。
不要做成不要做成成绩排名；不要过度医疗化解释；不要只发星星但不告诉为什么获得。
9月测试问题学生是否知道自己刚才获得了什么？是否愿意看成长记录？是否觉得这和减压/成长有关？

P0-P11映射

痛点 当前问题简述 对应Demo主线 Demo里如何验证
P0未成年人隐私、安全和危机边界必须先成立。Flow B，Flow A辅助看学生是否理解“AI会陪你，但明确风险时会优先保护你”；研究员模拟危机场景，不让学生扮演。
P1首屏双大卡片权重相当，无视觉主角。Flow A看学生 3 秒内是否能说出第一行动。
P2AI无全局持续入口，首页双入口认知存疑。Flow A、Flow B看切换模块后是否还能自然找到AI；是否理解不同AI入口的关系。
P3对话无情感记忆，关系从零开始。Flow B看轻量记忆是否提升信任，是否带来隐私不适。
P4无年级内容过滤，小学生看到选科志愿。横向原则Flow A/B/C 全部做年级文案差异，不单独做生涯规划Flow。
P5脑之岛成就不外溢，与首页和IP系统割裂。Flow C看训练完成后首页、AI、成长记录是否形成回流。
P6解忧信箱缺乏关闭反馈。Flow C可把“写完一封信”作为完成反馈变体，验证回执和被接住感。
P7换装系统与情绪成长割裂。Flow C奖励只做轻量露出，验证学生是否理解奖励来自成长行为。
P8星星商店激励闭环不清。Flow C看学生是否知道星星如何获得、为什么获得、可以去哪里用。
P9对话免责声明削弱信任感。Flow B对比“冷冰冰免责声明”与“温和边界说明”的接受度。
P10咨询Tab功能弱，容易变成空壳入口。Flow B把咨询入口收敛为AI/安全支持路径，不做复杂咨询预约。
P11任务缺乏情境化推送。Flow A、Flow C看今日小行动是否和情绪状态、完成记录相关。

MVP页面范围

20天内建议控制在 18-24 个关键画面。 不要追求每个功能都能点通，而是保证每条主线从入口、关键选择、反馈到结束完整可测。
模块 必做画面 可降级画面 不做画面
Flow A 首页启动 首页、今日状态选择、情绪打卡、AI接住、今日小行动、完成反馈。 成长记录只露出一条摘要。 完整任务中心、完整活动页。
Flow B AI伙伴 全局AI入口、三IP选择/切换、对话 2-3 轮、记忆提示、安全边界、下一步建议。 AI回复可用预设脚本，不接真实AI。 长历史、多轮复杂推理、真人咨询系统。
Flow C 完成反馈 训练/测评完成页、结果解释、奖励获得、成长记录、AI回应、首页回流。 训练过程可简化成 1 个可点击任务。 完整脑之岛地图、完整勋章墙、完整商店。
年级适配 同一主线至少准备“小学高年级/初高中”两套文案气质。 先不做真实账号识别，测试时由研究员切换版本。 完整内容标签后台。

成功标准

主线 最低成立标准 强改善标准 一票否决风险
Flow A 首页启动 多数学生能独立完成打卡并理解推荐理由。 首屏停顿明显少于 APK；被理解感、继续点击意愿均高于 APK。 学生觉得首页还是“很多东西不知道点什么”。
Flow B AI伙伴 多数学生能说出三IP差异，并愿意聊 2-3 轮。 学生主动表达“它像在回应我/记得我”；合规提示不破坏信任。 学生觉得记忆冒犯、边界吓人，或三IP只是换皮。
Flow C 完成反馈 多数学生知道自己做完了什么、获得了什么。 学生能理解训练/测评和情绪成长的关系，并愿意下次继续。 学生只记得拿奖励，不理解成长含义。

暂不做清单

本轮不做 原因 保留方式
完整星球大地图首页制作成本高，容易把测试引向视觉偏好。只保留局部世界观语言和IP反馈。
完整商店/换装经济系统20天内很难验证完整激励闭环。Flow C 只做一次奖励获得和去向提示。
家长端、教师端本轮是学生线下体验测试，角色会变复杂。只在安全边界文案中保留必要说明。
完整生涯规划与减压核心入口距离较远，且高低年级差异大。只把年级适配作为横向原则。
真实AI模型接入测试重点是体验路径，不是模型能力。用脚本化对话模拟关键回应。
危机对话让学生扮演未成年人测试不应诱导极端情绪表达。由研究员模拟，学生只评价界面是否清楚、可信、不过度惊吓。

Day 1-2行动清单

Day 1：范围对齐

  1. 团队确认本轮默认做 Flow A + Flow B，Flow C 作为条件主线。
  2. 确认目标测试学生分层：小学高年级、初中、高中是否都覆盖。
  3. 确认 demo 形式：Figma 可点击原型或 HTML demo。
  4. 确认 AI 对话采用脚本模拟，不等待真实模型。
  5. 确认危机响应只做研究员模拟，不让学生扮演危机场景。

Day 2：设计输入冻结

  1. 为每条主线写 1 句话价值主张。
  2. 列出每条主线的必做页面，不超过 8 个画面。
  3. 写出每条主线的测试成功标准。
  4. 准备竞品拆解清单：Finch、How We Feel、Wysa、Headspace、Elevate/Lumosity、Duolingo。
  5. 形成低保真流程图输入，进入 Day 3-5 竞品拆解和 Day 6-8 低保真。

本阶段结束时，你手里应该有

范围结论必做 Flow A、Flow B；时间允许做 Flow C。

页面边界总画面数控制在 18-24 个，不做完整系统。
测试指标首屏清晰度、任务完成、停顿点、被理解感、继续使用意愿、安全舒适度。
团队共识这不是改版交付，而是验证 2-3 条主线是否值得进入正式开发。
