Day 03-05 · 竞品Flow拆解与Demo设计输入

这份文件不是普通竞品分析，而是把真实行业标杆拆成可执行的 demo 设计输入：每条主线看谁、学什么、不学什么、最后转成哪些页面和交互。

本阶段目标

第 3-5 天只回答一个问题：这 3 条 Flow 应该怎么设计，才更可能在学生测试里赢过当前 APK？ 竞品不作为照搬对象，而是作为体验机制的参考：如何降低开始成本、如何让反馈有情绪价值、如何在安全边界内保持信任。
Day 3

核验与采样

确认竞品真实存在，建立每条 Flow 的标杆池，明确要看哪些页面和状态。

Day 4

机制拆解

拆“用户为什么愿意继续”：入口、引导、反馈、奖励、记忆、安全边界。

Day 5

转为输入

把竞品机制转成 AI星球 demo 的页面清单、交互规则和测试观察点。

Flow A

首页启动路径 · 竞品拆解

首页 → 今日状态 → 情绪打卡 → AI接住 → 今日小行动

主要看谁

竞品 要拆的机制 AI星球可借鉴点 不要照搬
Finch 先用轻量 check-in 开始，再把任务转成“照顾伙伴”的行动。 首页第一行动不要叫“任务”，而是“先看看今天的状态”；完成后让 IP 有即时回应。 不要完全宠物养成化，AI星球面向小学高年级到高中，过度低幼会损伤可信感。
How We Feel 用情绪词和颜色矩阵帮助用户找到更准确的表达。 情绪打卡不要只给“开心/难过”，可做 8-12 个低压力状态词，并允许“不知道”。 不要把矩阵做得太学术；线下 demo 先验证表达是否自然。
Headspace / Ebb AI理解用户状态后推荐合适练习。 今日小行动要有推荐理由：“因为你刚才选了紧绷，所以先来一个2分钟呼吸”。 不要推荐太多内容，Flow A 只给 1 个主推荐 + 1 个换一个。

Demo设计输入

首页结构

  • 首屏只突出一个主行动：今天感觉怎么样？
  • AI伙伴作为“接住你”的存在，而不是与打卡平级抢入口。
  • 功能入口后置，先完成状态识别再推荐。

情绪选择

  • 提供“有点烦、累、紧张、还好、开心、说不清”等低门槛词。
  • 允许跳过细节，避免逼学生解释。
  • 不同年级用不同文案密度，小学更直白，高中更克制。

AI接住

  • 先复述状态，再给一个小行动，不急着分析原因。
  • 语气要像“我听到了”，不是“你应该”。
  • 避免医疗化词汇，减少“评估、诊断、症状”。

今日小行动

  • 只推荐一个 1-3 分钟动作：呼吸、写一句话、轻小游戏、听一段音频。
  • 明确为什么推荐它。
  • 完成后要有回声：今天你已经照顾自己一下了。
Flow B

AI伙伴路径 · 竞品拆解

全局AI入口 → 三IP选择/切换 → 输入烦恼 → 伙伴回应 → 记忆/边界 → 下一步建议

主要看谁

竞品 要拆的机制 AI星球可借鉴点 不要照搬
Wysa AI对话通过结构化追问帮助用户表达，并明确不是专业医疗/危机服务。 三IP回应要有流程差异：小星陪伴，吉塔情绪稳定，罗素结构化分析。 不要把对话做成问卷，也不要用冰冷免责声明反复打断。
Headspace / Ebb AI先帮助整理想法，再推荐内容或练习。 每段对话结束都给“下一步”：继续聊、做个小练习、记录到成长本。 不要只把 AI 当入口搜索器，回复本身也要有被理解感。
988 / Crisis Text Line / Trevor 危机流程可预期：先确认安全，再连接真人/资源，并说明保密边界。 安全提示要温和透明：大多数内容会被尊重，明确危险时优先保护安全。 不要让学生在测试中扮演极端情绪；不要承诺绝对保密。

三IP在Demo里的角色定义

IP 适合场景 交互差异 一句话样例
小星 不知道怎么说、只是想有人陪。 少追问，多接纳；适合轻表达和日常陪伴。 “可以不用马上说清楚，我先陪你待一会儿。”
吉塔 焦虑、难过、压力明显，需要先稳住。 先确认感受，再给呼吸/落地练习；遇到风险时负责安全提示。 “听起来你现在很绷，我们先让身体慢一点。”
罗素 学习压力、目标混乱、选择困难、生涯方向。 帮用户拆问题、列选项、形成下一步计划。 “我们先把这件事分成可控制和暂时不可控制的两部分。”

Demo设计输入

全局入口

  • 在首页、训练完成页、成长记录页都能回到 AI 伙伴。
  • 入口文案根据场景变化：想聊聊、让吉塔帮你缓一下、问问罗素。
  • 不要只做底部 Tab；需要有情境化入口。

角色选择

  • 用“我现在想……”作为选择依据，而不是角色设定说明。
  • 选择后可以切换，降低选错压力。
  • 角色差异要在回复结构里体现，不只在头像和开场白里体现。

记忆设计

  • 只展示轻量记忆：“上次你说考试前会紧张”。
  • 给用户控制感：可以不记住、可以删除。
  • 不展示过细隐私，不制造“它偷看我”的感觉。

边界文案

  • 把“仅供参考”改成“我可以陪你整理想法，但不能替代老师、家长或专业帮助”。
  • 明确风险时使用保护性语言：“如果你可能伤害自己或别人，我会优先帮你找到安全支持”。
  • 危机入口要像支持，不像警告。
Flow C

完成反馈路径 · 竞品拆解

脑之岛/测评/放松训练 → 完成结果 → 低压力解释 → 奖励/成长记录 → AI回应 → 明天再来

主要看谁

竞品 要拆的机制 AI星球可借鉴点 不要照搬
Elevate 每日训练、个性化计划、能力维度和训练反馈。 脑之岛完成后给出“今天练到了什么能力”，并推荐下一次轻训练。 不要做成成绩压力，不要让学生觉得被排名。
Lumosity 基线测试、能力分区、进度追踪和历史趋势。 测评结果要转成“可理解的成长语言”，而不是只给分数。 不要在 demo 里做复杂曲线，先做单次完成后的价值感。
Duolingo 短任务、连贯奖励、成就、轻量提醒。 星星/徽章要和具体行为绑定：“完成一次专注训练，所以获得专注星”。 不要引入排行榜和强竞争，对减压产品不友好。
Finch 完成自我照顾后，伙伴获得成长和故事反馈。 奖励不要只是货币，而要让 IP 说出“我看见你完成了”。 不要让奖励喧宾夺主，心理成长比装扮更重要。

Demo设计输入

结果解释

  • 结果页先说“你完成了什么”，再说“这可能帮到什么”。
  • 避免给学生贴标签，比如“注意力差”。
  • 用成长语言：“今天练习了保持专注的能力”。

奖励闭环

  • 奖励来源要清楚：因为完成行为而获得，而不是随机掉落。
  • 奖励去向要清楚：可以用于伙伴装扮/星球收藏/成长记录。
  • 奖励反馈要轻，不要把减压体验变成刷任务。

成长记录

  • 记录不做成报告，做成“今天的一小步”。
  • 把情绪、行动、反馈连在一起。
  • 下次打开首页时能看见一点延续感。

AI回流

  • 完成后由对应 IP 说一句话，承接成果。
  • 允许继续聊：“要不要告诉我刚才做的时候哪里最难？”
  • 推荐明天再来，但不制造负担。

从竞品提炼出的设计原则

原则 解释 对应Demo要求
先接住，再推荐 学生不是来完成任务的，而是带着某种状态进入产品。 首页先问状态，AI先回应状态，再推荐行动。
低压力表达 好的情绪产品不会逼用户解释清楚，而是帮用户慢慢找到词。 情绪词要简单，允许“不知道/说不清/跳过”。
角色差异必须进入流程 IP人格不是设定表，而是不同场景下不同的工作方式。 小星、吉塔、罗素在追问、反馈、建议上必须不同。
安全边界要温和可预期 未成年人产品不能绝对承诺保密，也不能用恐吓式免责声明。 边界文案写成“我会尊重你，也会在明确危险时优先保护你”。
完成后必须有回声 没有反馈的训练像孤岛，有回应的训练才会成为成长记录。 每个完成页都要有结果解释、奖励原因、AI一句回应。
游戏化只服务成长感 奖励、星星、装扮可以增强动力，但不能压过减压和陪伴。 不做排行榜；奖励必须与情绪照顾/训练行为绑定。

文案样式输入

场景 推荐语气 避免语气
情绪打卡“今天比较像哪一种？” “说不清也可以。”“请准确描述你的情绪状态。”
AI回应“听起来你今天有点绷，我们先做个很小的调整。”“根据你的症状，建议你进行干预。”
记忆提示“我记得你上次说考试前会紧张，要不要接着聊？”“系统已记录你的心理问题。”
安全边界“大多数时候我会尊重你的分享；如果你可能伤害自己或别人，我会优先帮你找到安全支持。”“你的内容绝对不会被任何人知道。”
完成反馈“你完成了一次专注练习，今天的小进步已经记下来了。”“你的能力得分较低，请继续训练。”

3天工作安排

Day 3：竞品采样

  1. 按 Flow A/B/C 建立截图采集文件夹。
  2. 每个竞品只截 5-8 张关键页面，不收集无关功能。
  3. 截图命名使用“竞品_页面_机制”，例如 Finch_首页_checkin。
  4. 记录每张图解决的问题，而不是只评价好不好看。

Day 4：机制拆解

  1. 把每个竞品拆成入口、引导、反馈、奖励、回流。
  2. 为每条 Flow 写 3 条可借鉴机制。
  3. 标出不适合 AI星球的部分，防止误抄。
  4. 确认三IP在不同场景下的具体分工。

Day 5：设计输入冻结

  1. 冻结 Flow A/B 必做页面。
  2. 决定 Flow C 是完整闭环还是关键结果页。
  3. 写出情绪打卡、AI回应、边界提示、完成反馈的样本文案。
  4. 把本文件作为 Day 6-8 低保真流程图输入。
