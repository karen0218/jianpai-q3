Day 03-05 · 每个 Flow 一页竞品拆解

本文件是结果版，不是工作指引。每个 Flow 独立成页，重点只看三件事：竞品如何降低用户压力、如何引导下一步、如何给反馈。最后转译成 AI星球 demo 的具体设计输入。

Flow A 首页启动

首页 → 今日状态 → 情绪打卡 → AI接住 → 今日小行动

核心验证：学生打开 App 后，能不能在低压力下表达状态，并自然进入下一步。

本页参考竞品 Finch、How We Feel、Headspace / Ebb
重点看情绪表达、首个行动、个性化推荐。
竞品 怎么降低用户压力 怎么引导下一步 怎么给反馈 转成AI星球Demo
Finch 把自我照顾包装成“照顾伙伴”和轻量 check-in，不要求用户一上来解释复杂情绪。 从状态记录自然转向小目标、小任务、呼吸、反思等短行动，用户不用自己选一堆功能。 完成后用伙伴成长、鼓励语、奖励和进度回应用户，让“做了一件小事”被看见。 首页不要铺功能。先让小星问“今天比较像哪种状态”，完成后小星/吉塔给一句回应，再推荐 1 个今日小行动。
How We Feel 用情绪词、颜色和分类帮助用户找到表达，不把“说不清”当失败。 记录情绪后给用户看到模式、变化或下一步策略，让打卡不是孤立动作。 反馈重点不是评判，而是帮助用户理解“我现在是什么感觉，以及可能和什么有关”。 情绪打卡做成“低门槛词库”：有点烦、累、紧张、还好、开心、说不清。允许快速选择，不强迫长输入。
Headspace / Ebb 用陪伴式 AI 先承接用户表达，避免一开始就给训练清单或知识教育。 AI 根据当前状态推荐合适内容，把“聊天”导向呼吸、冥想、睡眠、放松等具体练习。 反馈由“理解 + 推荐理由”组成，让用户知道为什么现在适合做这件事。 今日推荐必须带理由：因为你刚才选择“紧张”，所以先来一个 2 分钟呼吸。只给一个主推荐，保留“换一个”。

Demo页面输入

  • 新版首页：一个主行动“今天感觉怎么样？”
  • 情绪词选择：6-12 个状态词，含“说不清”。
  • AI接住卡：复述状态 + 温和回应。
  • 今日小行动：1 个推荐 + 1 个换一个。
  • 完成反馈：记录到“今天的一小步”。

关键交互

  • 先状态，后功能。
  • 用户可以不输入文字也完成打卡。
  • 推荐内容跟随情绪词变化。
  • 完成后回到首页时出现“已被记住”的轻反馈。

测试观察点

  • 3 秒内是否知道先做什么。
  • 是否觉得打卡不费劲。
  • 是否感到 AI 回应了自己的状态。
  • 是否愿意点今日小行动。
Flow A 设计结论： 首页不要再让“情绪打卡”和“AI聊天”平级竞争。新版主线应该是：先用低压力状态选择帮学生开始，再让 AI 接住，最后只推荐一个小行动。
Flow B AI伙伴

全局AI入口 → 三IP选择/切换 → 对话回应 → 记忆/边界 → 下一步建议

核心验证：学生是否愿意和 AI 说、是否分得清三IP、是否觉得安全边界可信。

本页参考竞品 Wysa、Headspace / Ebb、988 Lifeline、Crisis Text Line、The Trevor Project
重点看AI陪伴、结构化支持、危机边界。
竞品/标杆 怎么降低用户压力 怎么引导下一步 怎么给反馈 转成AI星球Demo
Wysa 用聊天方式承接情绪，先让用户说一点点，再通过结构化追问慢慢展开。 根据用户表达进入不同支持路径，例如情绪整理、认知练习、呼吸练习或寻求专业帮助。 反馈不是简单安慰，而是“我理解 + 我们可以试一个方法”。 三IP不能只换头像。小星少追问多陪伴，吉塔先稳定情绪，罗素帮拆问题和计划下一步。
Headspace / Ebb AI 以陪伴和整理想法切入，降低用户进入心理内容的抵触。 对话后推荐具体练习，把“聊一聊”导向“做一个小动作”。 用个性化回应和推荐理由让用户感觉不是被丢进内容库。 每段 AI 对话结束都必须有行动卡：继续聊、做个小练习、记录到成长本。
988 / Crisis Text Line / Trevor 用清楚的“会发生什么”降低求助恐惧，说明用户不是立刻被审判或暴露。 当风险升高时，路径从 AI 陪伴转向安全支持、可信成年人、热线/文字支持等更合适的帮助。 反馈重点是确认安全和告诉用户下一步会怎样，避免神秘感和恐吓感。 边界文案改成温和透明：多数分享会被尊重；如果你可能伤害自己或别人，我会优先帮你找到安全支持。

三IP差异如何落到Flow里

IP 降低压力 引导下一步 反馈方式
小星 允许说不清，不急着追问原因。 引导用户说一句、选一个感受、或先待一会儿。 “我听到了，我会陪你慢慢说。”
吉塔 先稳定身体和情绪，不讨论大道理。 引导呼吸、落地练习、找安全支持。 “我们先让身体慢下来，安全最重要。”
罗素 把混乱问题拆小，减少失控感。 引导列选项、分轻重缓急、定一个可执行步骤。 “这件事可以先分成两部分，我们先处理能控制的。”

Demo页面输入

  • 全局 AI 入口：不同页面都能叫出伙伴。
  • 三IP选择页：用场景选人，而不是设定说明。
  • 对话页：每个 IP 一组 2-3 轮脚本。
  • 轻量记忆：可保留、可关闭、可删除。
  • 边界提示：温和安全说明。

关键交互

  • 角色可切换，降低选错压力。
  • AI 回复后必须给下一步，不只聊天。
  • 记忆提示不能过细，避免侵犯感。
  • 危机场景由研究员模拟，不让学生扮演。

测试观察点

  • 是否能说出三IP差异。
  • 是否愿意真实表达一点点。
  • 是否觉得记忆安心或不舒服。
  • 是否理解安全边界，不被吓到。
Flow B 设计结论： AI伙伴的重点不是“更像真人聊天”，而是让学生知道找谁、怎么说、说完能得到什么帮助，以及什么情况下会进入安全支持。
Flow C 完成反馈

脑之岛/测评/放松训练 → 结果解释 → 奖励/成长记录 → AI回应

核心验证：学生做完一件事后，是否知道自己获得了什么，并愿意下次继续。

本页参考竞品 Elevate、Lumosity、Duolingo、Finch
重点看训练反馈、成长记录、轻量激励。
竞品 怎么降低用户压力 怎么引导下一步 怎么给反馈 转成AI星球Demo
Elevate 把训练拆成短时、每日、可完成的小练习，避免用户面对一整套复杂训练。 训练后继续推荐下一项或明日计划，让用户知道不用自己规划。 反馈集中在能力维度和进步感，告诉用户今天练到了什么。 脑之岛完成后不要只显示完成。要告诉学生“今天练习了专注/记忆/反应中的哪一种”。
Lumosity 通过训练计划和能力分类减少“我该练什么”的选择压力。 用进度和表现趋势引导继续训练，让单次任务成为长期计划的一部分。 用可视化和解释把分数转成用户能理解的表现变化。 测评/训练结果不要只给分数。转成低压力解释：“这次你完成了一次注意力热身”。
Duolingo 每次任务很短，失败成本低，奖励即时出现，用户不会觉得要投入很久。 通过 streak、经验、下一课引导用户明天回来。 用即时奖励、成就、鼓励动效强化“我完成了”。 星星奖励要绑定原因：“完成一次放松练习，获得 3 颗放松星”。不做排行榜。
Finch 把自我照顾转成伙伴成长，用户不是在被考核，而是在照顾自己和伙伴。 完成后把用户带回伙伴、故事、装扮或下一项小目标。 反馈带情绪价值：伙伴回应你、成长一点点、奖励有温度。 完成页由小星/吉塔回应：“我看到你刚才完成了那 2 分钟，这已经是一小步了。”

Demo页面输入

  • 训练/测评完成页。
  • 结果解释：不贴标签，不排名。
  • 奖励获得：说明为什么获得。
  • 成长记录：今天的一小步。
  • AI回应：伙伴看见你的完成。
  • 首页回流：显示今日已完成状态。

关键交互

  • 结果先解释意义，再展示奖励。
  • 奖励不只是数值，要说明来源。
  • 成长记录不做报告，做轻量时间线。
  • 明天再来提示要温和，不制造压力。

测试观察点

  • 是否知道刚才完成了什么。
  • 是否理解这和减压/成长有关。
  • 是否知道星星/奖励从哪来。
  • 是否愿意下次再做一次。
Flow C 设计结论： 完成反馈不能只做“完成了 + 得分 + 奖励”。它要回答学生心里的问题：我刚才做这件事有什么用？它有没有被记住？下次我为什么还要回来？
