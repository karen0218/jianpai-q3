AI星球 / 减压星球 · 竞品材料评审

Claude 输出《核心竞品 Flow 深度分析》评审与整合建议

这份评审用于判断 Claude Code 输出的竞品分析是否能进入我们 Day 1-5 体系，以及该吸收哪些洞察、哪些需要降权或复核。

总体判断 这份 Claude 输出是有价值的，但它更像“竞品策略洞察稿”，不是我们现在需要的“每个 Flow 一页、可直接指导 20 天 demo 的设计输入”。建议吸收其中 4 个强洞察，但不要直接替代我们已做的 Day03-05 结果版。

1. 值得吸收的部分

可吸收

记忆 = 关系证明

它把 Replika 的记忆机制转译成“AI不是更聪明，而是更记得我”。这个洞察适合补强 Flow B，尤其是轻量记忆、对话开头历史钩子、成长档案。

可吸收

奖励必须有来源

它对 Finch 的理解很贴合 Flow C：奖励不是凭空给，而是和行为绑定。这个可以直接进入完成反馈和星星/换装轻闭环。

可吸收

首页是答案不是菜单

它对 Headspace 的判断和我们 Flow A 完全一致：首页应该告诉学生“现在最该做什么”，不是把所有入口平铺出来。

Claude洞察 适合放到我们哪份材料 怎么用
对话记忆钩子：上次你提到…… Flow B：AI伙伴路径 放到“轻量记忆设计”和“AI开场脚本”。但必须加“可查看、可删除、不默认过度记忆”。
换装/道具与具体行为绑定 Flow C：完成反馈路径 做 1 次奖励因果链：完成放松练习 → 获得放松星 → 可用于伙伴小物预览。
首页主角卡片逻辑 Flow A：Pad首页启动中枢 做“未打卡/已打卡”两态首页。未打卡主角是状态记录，已打卡主角是 AI接住或今日小行动。
AI能力边界说明替代免责声明 Flow B：安全边界 保留方向，但文案不能写“不会泄密”。要用未成年人安全边界语言。

2. 需要谨慎或降权的部分

点 为什么要谨慎 我的建议
Replika 作为“必参考” Replika 是 AI 情感伴侣，存在成人化、拟人依恋、关系边界模糊的产品气质。减压星球面向未成年人，不能照搬“情感伴侣”关系模型。 只学“可控记忆”和“连续关系感”，不学亲密陪伴、恋爱化、过度拟人依赖。
Woebot “唯一临床验证”表述 “唯一”这类绝对表述风险高，且原文提到的 18-28 岁研究不能直接等同 10-18 岁中学生。临床有效性也不等于产品可用性适合线下学生测试。 把 Woebot 从“临床权威背书”降为“结构化心理支持参考”。不要在团队汇报里写绝对化医学结论。
对话报告 + 情绪趋势图作为快赢 学生未必想看报告，且趋势图容易带来被监控感或成绩化。这个方向需要测试，不宜直接上升为 P0。 本轮 demo 只做“轻量总结”和“今天的一小步”，不要先做完整报告和趋势图。
打卡7天/对话3次/训练5次解锁道具 这个机制可行，但会把产品推向连续打卡激励。减压产品如果做重 streak，会增加新压力。 做“完成一次也被看见”的轻奖励，不强调断签、不做强连续压力。
缺少 How We Feel / Wysa / 988 Claude稿更偏 AI记忆、奖励、首页，但对情绪词表达、安全边界、危机支持流程覆盖不够。 仍保留我们原先的 How We Feel、Wysa、988/Crisis Text Line/Trevor 作为核心参考。
最重要的风险提醒 面向未成年人，Replika 只能作为“记忆机制参考”，不能作为“关系模型参考”。Woebot 只能作为“结构化支持参考”，不能被包装成“临床疗效证明”。这两点如果不收住，后面设计会跑偏。

3. 与我们现有 Day03-05 材料的关系

维度 Claude 输出 我们现有材料 结论
结构 按竞品逐个分析：Replika、Finch、Woebot、Headspace。 按 Flow 分析：Flow A 首页启动、Flow B AI伙伴、Flow C 完成反馈。 我们现有结构更适合 20 天 demo 执行。
强项 洞察鲜明，尤其是“记忆”“奖励来源”“Pad首页主行动”。 更完整覆盖情绪表达、安全边界、训练反馈、竞品走查。 Claude 适合补强洞察，我们适合做执行框架。
弱项 部分论断较满，且对未成年人安全边界不足。 可能没有 Claude 稿那样强烈的一句话洞察。 整合时用 Claude 的“金句”，保留我们的安全和落地边界。
对 Demo 的帮助 能帮助定义亮点：记忆、奖励、首页主角。 能直接指导页面节点和测试指标。 不要二选一，应该合并。

4. 推荐整合方案

整合原则 我们以“Flow A/B/C”为主结构，把 Claude 的强洞察塞进对应 Flow。不要改成以 Replika、Finch、Woebot、Headspace 为主的竞品报告，否则会离 20 天 demo 太远。
整合到哪个 Flow 吸收 Claude 哪个观点 整合后怎么写
Flow A：Pad首页启动中枢 Headspace：首页是答案，不是菜单。 首页主行动从“功能入口”改为“今天感觉怎么样”；完成打卡后，首页根据状态给唯一主推荐。
Flow B：AI伙伴 Replika：记忆是关系的证明。 AI对话开场加入轻量历史钩子，但只展示低敏信息，并提供“这次不记/管理记忆”。
Flow B：安全边界 Woebot：AI不必假装人类，明确分工能提升信任。 把“AI生成仅供参考”改成“我可以陪你整理想法，但不能替代老师、家长或专业帮助；明确危险时优先保护安全”。
Flow C：完成反馈 Finch：奖励要与行为绑定。 完成练习后显示“因为你完成了一次放松练习，获得 3 颗放松星”，并由 IP 说一句看见你的话。

5. 下一步动作

  1. 保留 Claude 原文件作为“竞品洞察参考稿”，不直接替代我们的 Day03-05 结果版。
  2. 把 Claude 的 4 个强洞察整合进我们的《Day03-05_竞品分析成果_Codex视角.html》。
  3. 在 Day01-05 封版表中，补一条最终判断：本轮 demo 的差异化关键词是“被记得、被接住、完成被看见”。
  4. 人工体验竞品时，重点验证 Claude 的高风险假设：学生会不会接受 AI 记忆？奖励会不会变成打卡压力？首页单主角是否真的更清楚？
我的建议 这份 Claude 输出可以用，但要“拆开用”。不要把 Replika、Woebot 直接作为产品方向；要把它们拆成小机制：记忆钩子、结构化支持、能力边界、行为奖励。这样才适合减压星球的学生场景。
