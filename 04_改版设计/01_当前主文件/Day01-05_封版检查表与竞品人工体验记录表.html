4. 竞品人工体验记录表

每个 App 只走首次体验 + 一个核心动作。不要深挖所有功能，避免被带散。已预填的内容来自 Claude Code 与 Codex 的整合判断，人工体验时可以直接修改。

竞品 重点体验路径 它怎么降低用户压力 它怎么引导下一步 它怎么给反馈 不适合借鉴
Finch Onboarding → check-in → self-care task → reward。 用“做一点也算数”和无失败惩罚降低压力；任务可以很小，不制造补签焦虑。 把下一步变成一个非常小的自我照顾动作，并让用户知道完成后会发生什么。 完成后有伙伴回应、视觉变化和奖励来源说明，形成“我的行动被看见”的感觉。 不照搬重宠物养成、强 streak、过度低幼视觉和复杂商店。
How We Feel Onboarding → emotion words → log feeling → insight/strategy。 用情绪词、分类和“说不清”选项降低表达门槛，避免一上来就让学生写长文本。 从状态选择自然进入原因、身体感受或一个短策略，路径不散。 记录完成后给轻量洞察或策略，而不是只显示“打卡成功”。 不做过重情绪矩阵，不让低年级用户被词汇复杂度卡住。
Wysa Onboarding → AI chat → prompt/reply → boundary/safety。 用预设话题、结构化追问和非评判语气让用户更容易开口。 每轮对话后给继续聊、练习、记录或寻求支持，不让对话停在情绪表达。 通过摘要、练习建议和边界说明反馈“我听到了，也知道下一步”。 不做成人心理工具式问卷，不让对话变冷、变诊断。
Headspace Onboarding → home recommendation → meditation/breathing → completion。 首页只给一个主行动，减少“我该点哪个”的选择压力。 根据当前状态推荐一个练习，完成后再给状态记录或下一步。 练习结束有完成感和状态回看，形成“做完后更清楚自己”的反馈。 不照搬成人冥想产品的冷静距离感，不把首页做成课程库。
Replika 首次对话 → 后续对话 → 记忆引用 → 关系连续感。 用户不必每次重新解释自己，AI 可以轻量引用过往状态。 用“上次你提到……”自然开启对话，再引导继续聊或做小行动。 通过记忆摘要、历史情绪钩子和可见记录反馈“它记得我”。 不学成人情感伴侣关系、暧昧表达、过度拟人依赖；记忆必须可控。
Woebot 情绪表达 → 情感确认 → 结构化支持 → 危机转介。 先承认感受，不急着建议；明确 AI 角色，减少“它是不是在假装真人”的不信任。 把表达收束到一个可执行下一步：呼吸、记录、认知重构或找支持。 对话结束给小结，高风险表达触发清晰的安全支持说明。 不夸大临床背书，不写绝对保密，不让学生在测试里扮演危机场景。
Daylio Mood log → activity tag → note optional → trend later。 首次记录很短，允许不写原因；先完成，再逐步补充更多信息。 用活动标签和可选备注引导补充，但不强迫用户解释清楚。 累计记录后再给趋势、模式和回看价值，让数据反馈晚一点出现。 不做强打卡压力，不在数据不足时给空洞洞察。
Duolingo Onboarding → first lesson → instant feedback → reward/streak。 任务极短、反馈即时，用户知道“现在只要完成这一小步”。 完成后马上给下一步，但下一步要可跳过，避免强迫连续。 用即时反馈和轻奖励制造完成感。 不学排行榜、损失厌恶、强连续打卡和失败惩罚。
Elevate / Lumosity 二选一：training → result → progress/next training。 训练结果不用排名压迫，而是解释“今天练到了什么能力”。 结果页给一个相关下一步：再练 1 分钟、查看成长记录或让 AI 解读。 用能力解释、趋势和进步语言替代单纯分数。 不做能力贴标签、排名、分数羞辱和考试感。

5. 按 Flow 汇总你的判断

Flow A

Pad 首页启动中枢

最值得借鉴：

Headspace 的“首页是答案不是菜单”、How We Feel 的低压力情绪表达、Daylio 的短路径打卡；同时用 Pad 空间承载 AI 伙伴、今日推荐、成长反馈和次级功能入口。

需要避开：

避开功能入口平铺、过重情绪矩阵、强打卡压力、单一打卡页，以及把 Pad 首页做得像手机小卡片列表。

对 AI星球 的修改：

首页未打卡时主卡突出“今天感觉怎么样”；右侧/侧栏露出 AI 伙伴，下方提供今日推荐、成长反馈和次级功能入口。打卡后由 AI 接住，再推荐一个今日小行动，并让首页状态更新。

Flow B

AI伙伴

最值得借鉴：

Replika 的轻量记忆钩子、Woebot 的能力边界、Wysa 的结构化下一步、危机支持产品的安全转介逻辑。

需要避开：

避开成人情感伴侣关系、绝对保密承诺、临床疗效夸大和让学生扮演危机场景。

对 AI星球 的修改：

三 IP 选择用场景而不是角色名来引导；对话要先确认感受，再给继续聊/练习/记录/找支持；记忆必须可见可控。

Flow C

完成反馈

最值得借鉴：

Finch 的“行为产生奖励”、Elevate/Lumosity 的能力解释、Duolingo 的即时反馈。

需要避开：

避开排名、能力贴标签、强 streak、复杂商店和只给分数不解释意义。

对 AI星球 的修改：

完成页必须说明“你刚刚练到了什么”和“奖励从哪里来”；首页或 AI 要回看这次完成，让模块之间连起来。
