Day 01-02 · 核心Flow定范围结果版

本文件是第 1-2 天的结果产出，不是工作指引。它把 20 天 demo 的范围、优先级、验证假设、页面边界和成功标准直接定稿，供团队对齐和后续低保真开工使用。

最终定稿

20 天 demo 采用“2 条必做 + 1 条条件做”的范围。 Flow A 和 Flow B 是本轮必须完成的高保真主线；Flow C 是保留价值验证的第三条主线，时间不足时只做关键闭环，不做完整模块。
必做 / 高保真

Flow A：Pad 首页启动中枢

Pad首页 → 今日状态中枢 → 情绪打卡 → AI接住 → 今日推荐/功能分流 → 小行动 → 首页回流。

解决“打开后不知道先做什么”的首屏问题。

必做 / 高保真

Flow B：AI伙伴路径

全局AI入口 → 三IP选择/切换 → 对话回应 → 记忆/边界 → 下一步建议。

验证产品差异化核心：学生是否愿意和 AI 伙伴说。

条件做 / 中高保真

Flow C：完成反馈路径

脑之岛/测评/放松训练 → 结果解释 → 成长记录 → AI回应。

验证做完后有没有价值感和继续使用理由。

范围决策表

候选Flow 本轮结论 为什么做/不做 本轮做到什么程度
Pad 首页启动中枢 + 情绪打卡 + AI接住 + 今日推荐分流 必做 这是学生打开产品后的第一体验，直接影响是否愿意继续。也最适合和当前 APK 首屏做对照。 高保真完整闭环，从首页到完成反馈全部可点通。
AI伙伴三IP + 对话 + 安全边界 必做 AI伙伴是产品差异化核心，也是信任和合规风险最高的地方，必须在学生测试前验证。 高保真关键路径，用脚本模拟 2-3 轮对话，不接真实模型。
脑之岛/测评/放松完成反馈 条件做 它决定产品有没有持续使用理由，但相对 Flow A/B 优先级低。 中高保真闭环。若时间不足，只做完成页、成长记录、AI回应、首页回流。
完整星球地图首页 不做主线 视觉和资产成本高，容易让测试变成“喜欢哪个皮肤”。 只把世界观/IP作为局部反馈语言，不做大地图导航。
完整商店/换装/星星经济 不做完整系统 20 天内很难验证完整经济闭环，且容易偏离减压主线。 只在 Flow C 中做一次奖励获得和奖励去向提示。
家长端/教师端/生涯规划完整流程 不进入本轮 本轮目标是学生线下体验测试，跨角色和长周期决策会稀释主线。 只保留年级适配和安全边界原则。
Flow A 必做

Pad 首页启动中枢

Pad首页 → 今日状态中枢 → 情绪打卡 → AI接住 → 今日推荐/功能分流 → 小行动 → 完成反馈 → 首页回流

目标用户刚打开 App、不知道自己要做什么的学生；尤其是“有点烦、累、紧张、但说不清”的轻压力用户。
要解决的问题当前 APK 首屏情绪打卡和 AI 伙伴并列，Pad 屏幕空间没有形成“今日状态中枢”，用户既不知道先做什么，也看不到其他功能与自己当下状态的关系。
验证假设如果首页用 Pad 空间组织成“一个主行动 + 多个状态相关区域”，学生既能 3 秒内知道先从哪里开始，也能感到产品内容足够丰富、不是单一打卡工具。
用户价值不用理解产品全貌，也能在 30 秒内开始；同时能看到 AI、今日推荐、成长记录和功能入口都围绕“我今天的状态”服务。
设计重点主行动唯一，信息区丰富：左侧/中心为今日状态主卡，右侧为 AI 伙伴状态入口，下方为今日推荐、成长反馈和次级功能入口。
不做内容不做完整任务中心，不做复杂活动推荐，不做长问卷式情绪测评；也不把 Pad 首页做成所有功能平铺的大菜单。

Pad 首页中枢结构

首页区域 作用 呈现原则
今日状态主卡承接第一行动：今天感觉怎么样。面积最大、层级最高，支持直接点情绪词或开始记录。
AI 伙伴侧栏让小星/吉塔/罗素常驻，不再只是 Tab 入口。根据状态变化文案，但不抢主行动。
今日推荐区把放松训练、脑之岛、记录等功能变成“和我有关的下一步”。只给 1 个主推荐 + 1-2 个次级推荐，避免功能平铺。
成长反馈条展示今日已记录、已完成、获得星星或被 AI 看见。轻量显示，不做复杂成就墙。
次级功能入口保留 Pad 应有的信息量和探索空间。课程、测评、训练、信箱等入口可见，但视觉权重低于今日状态。
Flow A 定稿结论： 它是新版 Pad 端体验骨架，必须优先做高保真。9 月测试时，它不只证明新版是否更容易开始，也要证明首页是否能承载“状态、AI、推荐、成长、功能入口”的中枢价值。
Flow B 必做

AI伙伴路径

全局AI入口 → 三IP选择/切换 → 输入烦恼 → 伙伴回应 → 轻量记忆 → 安全边界 → 下一步建议

目标用户愿意尝试和 AI 说话的学生，以及想倾诉但对真人表达有压力的学生。
要解决的问题当前 APK 中 AI 入口缺少持续存在感，三IP差异没有形成清晰工作流，对话记忆和免责声明也会影响信任。
验证假设如果三IP以不同场景和不同回应方式出现，并提供可控的轻量记忆与温和安全边界，学生会更愿意表达，也更能理解 AI 伙伴的价值。
用户价值学生知道“现在该找谁”，说完之后能得到陪伴、稳定情绪或拆解问题的下一步。
设计重点三IP按场景选择；回复结构体现差异；记忆可控；安全边界透明但不吓人。
不做内容不接真实大模型，不做复杂长对话，不做真人咨询预约闭环，不让学生扮演危机场景。
Flow B 定稿结论： 它是产品差异化核心，必须与 Flow A 同等优先。9 月测试时，它负责证明学生是否愿意把 AI 当成可信伙伴，而不是普通问答入口。
Flow C 条件做

完成反馈路径

脑之岛/测评/放松训练 → 完成结果 → 低压力解释 → 奖励/成长记录 → AI回应 → 首页回流

目标用户完成训练、测评、放松练习或写信后，需要知道“这件事有什么用”的学生。
要解决的问题当前 APK 的脑之岛、解忧信箱、星星商店、换装奖励较割裂，完成动作没有外溢到首页、AI和成长叙事。
验证假设如果完成后能看到低压力结果解释、奖励原因、成长记录和 AI 回应，学生会更理解这件事和减压/成长有关，也更愿意下次继续。
用户价值做完不是结束，而是被看见、被记录，并形成“我今天完成了一小步”的成长感。
设计重点结果解释不排名；奖励说明来源；成长记录轻量；AI回应承接完成动作。
不做内容不做完整脑之岛地图，不做完整勋章墙，不做完整星星商店，不做排行榜。
Flow C 定稿结论： 如果团队只能做两条，先放弃 Flow C 的完整高保真；但至少建议保留一个“完成反馈页 + 成长记录 + AI回应”的轻闭环，用来验证持续使用价值。

P0-P11覆盖结果

痛点 进入本轮吗 由哪个Flow承接 本轮验证方式
P0 未成年人安全/隐私/危机边界进入Flow B验证安全边界文案是否可信、不吓人；危机场景由研究员模拟。
P1 首屏无视觉主角进入Flow A看学生 3 秒内是否知道先做什么，并能否理解首页不是单一打卡页，而是今日状态中枢。
P2 AI无全局持续入口/双入口认知进入Flow A、B看学生是否能在不同页面找到 AI，并理解入口关系。
P3 对话无情感记忆进入Flow B验证轻量记忆是否提升信任，是否引发不适。
P4 年级内容过滤不足横向进入Flow A、B、C同一主线至少区分小学高年级/初高中语气，不单独做生涯Flow。
P5 脑之岛成就不外溢条件进入Flow C看训练完成后是否能回到首页、AI和成长记录。
P6 解忧信箱缺少关闭反馈条件进入Flow C变体可作为“完成反馈”变体，不单独做完整写信Flow。
P7 换装与成长割裂轻量进入Flow C只验证奖励是否来自成长行为，不做完整装扮系统。
P8 星星激励闭环不清轻量进入Flow C只验证星星来源和去向是否清楚。
P9 免责声明削弱信任进入Flow B验证温和边界说明是否比冷免责声明更可接受。
P10 咨询Tab功能弱部分进入Flow B不做咨询Tab完整改造，只验证安全支持入口。
P11 任务缺乏情境推送进入Flow A、C看今日小行动是否和状态/完成记录有关。

MVP页面范围

Flow 必做页面/状态 页面数量建议 保真度
Flow A：Pad 首页启动中枢 Pad 首页、今日状态主卡、AI伙伴侧栏、今日推荐区、次级功能入口、情绪打卡确认、AI接住卡、今日小行动、行动完成反馈、首页回流。 6-7 个 高保真

Flow C：完成反馈 训练/测评完成页、结果解释、奖励获得、成长记录、AI回应、首页回流。 5-6 个 中高保真，可降级
总量控制： 完整做 3 条时控制在 18-22 个关键画面；只做 2 条时控制在 13-16 个关键画面。

成功标准

Flow 最低成立 强改善 失败信号
Flow A：Pad 首页启动中枢 多数学生能独立完成打卡，并理解系统为什么推荐这个小行动。 比当前 APK 首屏停顿更少，被理解感和继续点击意愿明显更高。 学生仍然说“不知道先点哪个”，或觉得推荐像普通任务列表。
Flow B：AI伙伴 多数学生能说出三IP差异，并愿意聊 2-3 轮。 学生主动表达“它像在回应我/记得我”，且不被安全边界文案劝退。 学生觉得三IP只是换头像，或记忆/边界让人不舒服。
Flow C：完成反馈 多数学生知道自己完成了什么、获得了什么。 学生理解训练/测评和减压成长有关，并愿意下次继续。 学生只记得拿奖励，不理解成长含义，或觉得像成绩考核。

Day 1-2冻结清单

已经冻结

  • 本轮 demo 不做完整新版本，只做 2-3 条核心 Flow。
  • Flow A、Flow B 必做，且要高保真。
  • Flow C 条件做，优先保留完成反馈轻闭环。
  • AI 对话用脚本模拟，不依赖真实模型。
  • 危机响应不让学生扮演，由研究员模拟。
