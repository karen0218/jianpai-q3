Day 03-05 · 竞品分析整合定稿版

本文件整合 Claude Code 与 Codex 的竞品分析成果。最终结构以 Flow A/B/C 为主，因为本阶段目标不是完成一份泛竞品报告，而是为 20 天 demo 的页面设计、交互文案和测试验证提供统一依据。

最终结论 本轮竞品分析采用“Flow 主结构 + 竞品机制分层”的方式：用 Headspace/How We Feel/Finch/Daylio 解决 Pad 首页启动中枢、低压力情绪开始和状态相关推荐，用 Wysa/Headspace Ebb/988 补足 AI 支持与安全边界，用 Replika/Woebot 补强 AI 记忆和能力边界，用 Finch/Elevate/Lumosity/Duolingo 解决完成反馈和轻激励。所有竞品机制最终都必须回到 Flow A/B/C，不再新增平行主线。

1. 两套材料如何分工

材料类型 主要价值 在本定稿中的位置
Claude Code 策略稿 洞察强，适合回答“为什么要这样设计”。例如 Replika 的记忆、Finch 的奖励绑定、Headspace 的首页主CTA。 作为“补充洞察层”进入各 Flow。
Claude Code 深度分析 更适合后续画原型时查具体流程、状态、文案、反馈细节。 作为 Day 6-8 低保真和 Day 9-13 高保真的执行素材。
Codex Flow 分析 按 Flow A/B/C 组织，更适合 20 天 demo 的范围、页面结构和测试指标。 作为本定稿的主骨架。
人工竞品体验记录 补充真实手感：首次打开是否有压力、文案是否自然、反馈是否真有价值。 作为 Day 1-5 最终封版前的校验，不改变主结构。

2. 竞品池定稿

竞品/标杆 定稿角色 用于哪个 Flow 学什么 不学什么
Headspace / Ebb 首页主行动 + AI内容推荐 Flow A、Flow B 首页是答案不是菜单；AI承接表达后推荐练习。 不照搬成人冥想产品的冷静距离感。
How We Feel 低压力情绪表达 Flow A 用情绪词、分类和“说不清”帮助用户开始表达。 不做过重情绪矩阵，不让低年级用户认知负担过高。
Finch 陪伴式自我照顾 + 奖励来源 Flow A、Flow C 完成小行动后被伙伴看见；奖励必须绑定具体行为。 不做重宠物养成，不把产品做低幼。
Wysa AI心理支持与结构化追问 Flow B 从表达进入自助练习、下一步建议和安全支持。 不做成人心理工具式问卷，不让对话变冷。
988 / Crisis Text Line / Trevor 危机支持与保密边界 Flow B 说明“什么情况下会找支持”和“接下来会发生什么”。 不让学生扮演危机场景，不做恐吓式提示。
Replika AI持续记忆机制 Flow B 对话开头的轻量记忆钩子，让用户感到“它记得我”。 不学成人情感伴侣模型，不做拟人依赖。
Woebot AI能力边界与结构化支持 Flow B 明确 AI 不是人类/不替代专业帮助，反而提升可信度。 不夸大临床背书，不把研究结论直接套到 10-18 岁学生。
Daylio 快速情绪打卡执行细节 Flow A 用极短路径完成记录，降低开始成本。 不做强 streak 压力，不把打卡变成任务负担。
Elevate / Lumosity 训练结果解释 Flow C 告诉用户“你练到了什么能力”，让训练有意义。 不做排名、分数压迫和能力评判。
Duolingo 短任务与即时反馈 Flow C 任务短、反馈快、完成后有下一步。 不学排行榜、强连续打卡和损失厌恶。

3. 6 条设计策略定稿

01

首页是答案，不是菜单

首屏只给一个主行动。未打卡时先问状态，已打卡后给一个与状态相关的下一步。

02

允许说不清

情绪表达必须低压力，“说不清”是合法状态，不是异常输入。

03

AI要接住，也要收束

AI先回应感受，再把用户带到继续聊、练习、记录或安全支持。

04

记忆要可控

AI可以记得轻量情绪历史，但必须让学生知道记住了什么，并能不记或删除。

05

完成必须被看见

训练、打卡、对话做完后要被首页、AI或成长记录接住，避免模块孤岛。

06

安全边界先于情绪承诺

不能承诺绝对保密。要温和说明：大多数分享被尊重，明确危险时优先保护安全。

Flow A 必做 / 高保真

Pad 首页启动中枢

Pad首页 → 今日状态主卡 → 情绪打卡 → AI接住 → 今日推荐/功能分流 → 小行动 → 首页回流

主参考 补充参考 降低压力 引导下一步 反馈机制 Demo 落地
Headspace、How We Feel、Finch Daylio 用“今天感觉怎么样”作为主入口，同时让 Pad 首页保留 AI、推荐、成长和功能探索空间；允许说不清。 先用主卡承接状态，再把其他功能包装成“今日相关”的 1 个主推荐 + 1-2 个次级推荐。 AI先复述状态，再解释为什么推荐这个小行动；首页回流后展示今日已记录、AI已接住、推荐待完成或已完成。 做 Pad 首页五区结构：今日状态主卡、AI伙伴侧栏、今日推荐区、成长反馈条、次级功能入口；未打卡/已打卡/完成后三态都要有。
Flow A 定稿判断 Flow A 不再定义为单一情绪打卡链路，而是 Pad 首页启动中枢。新版首页应该主行动清楚、信息承载丰富：用“状态识别”作为第一步，让 AI 伙伴成为承接者，让今日推荐把脑之岛、放松训练、记录等功能组织成和用户状态有关的下一步。
Flow B 必做 / 高保真

AI伙伴路径

全局AI入口 → 三IP选择/切换 → 对话回应 → 轻量记忆 → 安全边界 → 下一步建议

主参考 补充参考 降低压力 引导下一步 反馈机制 Demo 落地
Wysa、Headspace/Ebb、988/CTL/Trevor Replika、Woebot、Character.AI 用“找谁更适合现在的我”替代“请选择AI角色”；允许只点预设话题，不强迫输入长文本。 小星陪伴，吉塔稳定，罗素拆解；每轮对话后给继续聊/练习/记录/找支持。 轻量记忆钩子 + 可控记忆 + 温和能力边界说明。 三IP选择页加入场景标签和语气示例；对话页做 2-3 轮脚本；记忆提示必须可记/不记/删除。

三 IP 工作流定稿

IP 角色定位 回复结构 下一步
小星 陪伴型入口，适合“说不清”“想被陪陪”。 接纳 + 陪伴 + 轻问题。 继续说一句 / 记录今天状态。
吉塔 情绪稳定与安全支持，适合焦虑、难过、紧绷。 确认感受 + 身体稳定 + 安全边界。 呼吸练习 / 找可信成年人 / 安全支持。
罗素 结构化分析，适合学习压力、选择困难、目标混乱。 复述问题 + 拆小 + 形成一步计划。 列计划 / 做训练 / 记录下一步。
Flow B 定稿判断 AI伙伴的竞争力不是“像真人”或“更会聊天”，而是学生能感到：它知道我现在该找谁，它记得一点点我的状态，它知道什么时候应该找更安全的支持。
Flow C 条件做 / 轻闭环

完成反馈路径

脑之岛/测评/放松训练 → 结果解释 → 奖励来源 → 成长记录 → AI回应

主参考 补充参考 降低压力 引导下一步 反馈机制 Demo 落地
Finch、Elevate、Lumosity Duolingo 结果不排名、不诊断、不说能力差，只说“今天练到了什么”。 完成后引导查看成长记录、让 AI 回应、明天做 1 分钟也可以。 奖励必须说明来源；IP 要说一句“我看见你完成了”。 只做轻闭环：完成页 + 结果解释 + 奖励来源 + 成长记录 + AI回应 + 首页回流。
Flow C 定稿判断 完成反馈不是“完成了 + 得分 + 奖励”，而是“我知道自己刚才做了什么、这有什么意义、它被谁看见了”。Flow C 不做完整脑之岛和完整商店，只验证完成被看见。

7. 明确不照搬清单

竞品来源 不照搬什么 原因 替代方案
Replika 成人情感伴侣关系、过度拟人依赖。 未成年人产品要控制依恋和边界风险。 只学轻量记忆钩子和可控记忆。
Woebot 过度临床背书或医学化表达。 研究证据不能直接套用到本产品和 10-18 岁学生测试。 只学结构化支持和能力边界说明。
Finch 重宠物养成和过度低幼视觉。 高中生可能不买账，且会削弱可信感。 保留“伙伴看见我”和奖励来源。
Duolingo / Daylio 强 streak、排行榜、断签压力。 减压产品不能制造新的焦虑。 做低压力连续感：回来看看也很好。
Elevate / Lumosity 排名、分数压迫、能力贴标签。 学生会把训练误解为考试和评价。 把结果转译成成长语言：今天练习了专注。
