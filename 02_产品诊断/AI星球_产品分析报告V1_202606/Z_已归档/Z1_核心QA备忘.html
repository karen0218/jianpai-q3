Q1 年段适配是三套前后端，还是一套系统？ ▶
2026.06.11
A：一套系统，一套代码，按 phaseCode 渲染不同视图层。
更接近同一个网站做了三个皮肤，而不是三个不同的产品。

APK 里已经有 phaseCode（小学/初中/高中）和 gradeCode（具体年级），登录时已知，只是现在没有用它来过滤内容。
后端：一次「内容打标」 所有内容、任务、测评模板打上适用年级标签；API 返回内容时带上 phaseCode 过滤参数。这不是三套 API，是一套 API 加了一个筛选维度。
前端：三套「视图配置」 首页布局配置按 phaseCode 读取不同配置文件；组件本身复用，只是排列密度、文字描述、功能入口的可见性不同。
技术实现四步：登录时读 phaseCode → 读视图配置 JSON → 组件加 density prop → 后端内容打标过滤。改动范围：写三份 JSON + 给内容打标签，不需要重写业务逻辑。

最值得关注的点：年级衔接。初升高时罗素能引用吉塔对话里的兴趣记录——这是护城河靠时间积累的核心逻辑。

Q14 D2 保守版原型里学龄分段是怎么体现的？小学/初中/高中首页有什么不同？ ▶
2026.06.16
A：D2 保守版加了可交互的年级切换条（🌱小学 / 🌀初中 / 🎯高中），切换后整个信息流实时变换。

🌱 小学（phaseCode=primary）
情绪签到 → 5个表情脸（😊😐😓😔😤），不是6个邻居卡片；主导IP换成小星；任务用「今日探险」游戏化语言；推荐内容是记忆翻牌游戏/情绪涂色，没有测评/生涯类内容。

🌀 初中（phaseCode=middle）
当前默认视图，全部6个邻居选择（鸭梨/阿眠/小刺/拖拖/小炎/皮皮）；吉塔主导；任务含情绪压力测评+呼吸法；无生涯/高考内容。

🎯 高中（phaseCode=high）
顶部新增高考倒计时徽章（可关闭）；任务由罗素主导，含「学业压力专项测评」+「选科兴趣初测」；推荐区新增「选科分析：文理倾向」；IP悬浮球默认换成罗素。

技术实现：一套组件，用 class（g-primary / g-middle / g-high）+ JS切换 gshow 控制显隐，与B3方案完全一致。

Q11 导航为什么从6个 Tab 缩减到3个？任务、探索、朋友圈去哪了？ ▶
2026.06.16
A：三个 Tab 被整合进首页，不是删除而是降层级。

任务→ 变成「吉塔今日安排」AI推送卡片，签到后吉塔根据情绪状态自动推送当日2-3个任务。

Q17 脑之岛内部体验完整，勋章/解锁都有，A5 的 P5 痛点指的是哪里断了？ ▶
2026.06.16
A：脑之岛内部体验完整，痛点在于成就不外溢——两套系统互相割裂。

脑之岛内部已有的：引导弹窗（新用户首次进入有 Onboarding）、勋章墙（完成训练获得勋章）、解锁机制（带锁/解锁状态）、任务卡——内部体验本身没有问题，不存在新用户迷路，也不缺少情感反馈。

真正缺的是跨模块串联：
脑之岛训练完成后，吉塔/罗素不感知；首页不回应；成就停在脑之岛内部，不形成整体成长叙事的闭环。用户完成训练后回到首页——一切如常，感知不到「这次训练对我的心理健康有什么意义」。
挑战版是这个痛点的解法：从星球地图进入脑之岛时罗素出现引导；完成训练→迷雾散开→邻居解锁，脑之岛成就第一次真正传递给整体成长叙事，形成「做了这件事 → 世界变了」的正向反馈闭环。

Q13 挑战版用了两个地图，会不会让 app 太「游戏化」失去心理工具感？ ▶
2026.06.16
A：挑战版的设计决策恰好解决了这个问题，而不是加重它。

问题的根源：保守版才是两图并列的（首页星球面板 + 脑之岛Tab训练地图），两套地图逻辑同屏并存，用户需要理解两种「地图」的不同含义，认知负担更高。

挑战版的解法：层级化取代并列。减压星球地图是第一层（情感世界），脑之岛是第二层（点进Zone才进入训练地图）。两层视觉语言完全不同：
减压星球首页：手绘柔和治愈风，低对比，暖色调（紫/绿），情感优先
脑之岛内部：功能化UI，高对比，冷色调（蓝/靛），进度条/徽章/关卡编号，目标感优先
进入脑之岛的「穿越感」转场动效本身就是信号——画风切换告诉用户换了场景，不会让整体 app 戏化过度。

Q3 APK 截图里小星是白色的，但 B2 规范里小星主色是紫色，矛盾怎么解释？ ▶
2026.06.11
A：不矛盾。小星角色本体是白色，B2 里的紫色是「小星在 UI 系统里的品牌色」。

小星外形：白色棉花糖身形，头顶黄色星星发饰，颈间粉色围巾——这是固定的角色造型，不会变。

B2 分配给小星的 #A855F7 紫色，是她在界面里的功能色：悬浮球背景色、对话气泡边框、相关按钮的强调色。就像米老鼠的造型是黑白，但迪士尼用红色代表他的品牌——角色色和UI品牌色是两个维度。
实际分工小星=通用AI伴侣（#A855F7紫）；吉塔=情绪疗愈（#10B981绿）；罗素=逻辑生涯（#0EA5E9蓝）。三色系在界面中区分三IP的功能区域。

Q5 三个 IP 分别管什么？切换 IP 是用户主动操作还是系统自动切换？ ▶
2026.06.11
三IP职责分工：
小星🤍（通用AI）默认陪伴，什么话题都能聊；情绪中性，适合探索和日常互动；入口最低，是新用户第一个接触的IP
吉塔🐼（情绪疗愈）专门处理负向情绪；选了「鸭梨/小炎/阿眠」类邻居时吉塔主导；聊愈室触发场景以吉塔为主；话语风格：先接住情绪，再问问题
罗素🐳（逻辑/生涯）高中版默认主导IP；生涯规划、选科、学业压力场景；话语风格：把模糊的问题拆成可回答的具体问题
切换方式：两种并存。用户可在对话界面右上角设置里主动切换（A7 Flow4）；系统也会根据情绪签到结果和年级，在首页悬浮球默认显示推荐IP——高中生默认显示罗素，初中生默认吉塔。

Q6 如果学生在 AI 对话里说了危险的话，系统怎么处理？危机干预分几级？ ▶
2026.06.11
三级危机干预协议：
L1 · 情绪关注检测到负向情绪词汇（「好累」「不想去学校」）→ IP回复切换为更温柔的接纳语气，推送减压工具；不通知任何人，纯在对话内处理
L2 · 预警提醒检测到中风险关键词（「没意思」「消失」「不想活」但无明确意图）→ IP先稳定情绪，同时在心理老师端触发黄色提醒；老师可选择主动联系或继续观察
L3 · 紧急干预检测到高风险表达（明确的自我伤害意图）→ IP立即给出危机热线（12320/北京心理危机研究与干预中心），同时通知心理老师和家长；APK代码已有 crisisLevel 字段
关键设计原则：L1不泄露隐私；L2告知老师但不告知家长（除非情况升级）；L3家长知情是最后手段，避免学生因担心被告知而不敢说真话。

Q15 E1 世界观圣经和 E2 产品体验设计提案什么关系？先看哪个？ ▶
2026.06.16
A：E1 是「是什么」，E2 是「怎么做」。先看 E1，再看 E2。

E1 定义底层：减压星球的宇宙起源、三IP的人格与定位、六大区域的设定、鸭梨等邻居角色的来历——这是世界观圣经，给设计和运营提供统一的叙事基础。

如果只能看一份：E2 更直接——它把 E1 的概念直接翻译成了可以开发的功能需求。

Q16 挑战版里减压星球和脑之岛两个地图怎么融合？产品/视觉/交互三个维度分别怎么处理？ ▶
2026.06.16
A：两个地图的答案不是选一个删一个——减压星球做导航层，脑之岛做内容层。

产品维度：层级化，不并列
减压星球 = 第一层（情感导航，找IP，探索区域）。脑之岛 = 第二层（点进Zone才进入训练关卡）。脑之岛不消失，但不再是独立Tab——它是星球上的一座岛，有自己的内部地图。E2提案四的叙事层方案同步落地：完成训练→迷雾散开→解锁迷雾森林居民，两套系统形成闭环。
视觉维度：风格拉开 + 穿越感转场
减压星球首页：手绘柔和治愈风，暖紫/绿色调，动画轻柔，情感优先。脑之岛内部：功能化UI，冷蓝/靛色调，进度条/徽章/关卡编号，目标感优先。进入脑之岛时「穿越感」转场（从星球地图飞入某个区域），画风切换本身是场景切换信号。B1诊断的4个蓝色混用问题反而可以利用——两层用不同色温，颜色做场景信号。
交互维度：强化脑之岛情感串联
A5诊断P5：脑之岛训练完成后缺乏情感层反馈，用户感知不到训练与心理健康主线的关系（脑之岛导航本身清晰，不存在新用户迷路问题）。从星球地图点击进入脑之岛时，罗素主动出现引导；完成训练→迷雾散开，与外层邻居解锁形成「做了这件事→世界变了」正向反馈。

Q9 这个产品怎么打分？心理健康类 app 有哪些特有的风险点？ ▶
2026.06.11
产品综合评分（基于 APK 现状）：6.2 / 10
优势项语音对话技术扎实（科大讯飞核心能力）；三IP差异化设计有原创性；B2B采购路径可行；phaseCode/gradeCode基础设施已有
短板项世界观停留在心理剧场一个入口，未渗透其他功能；首页信息密度高但无主线引导；设计系统碎片化（B1的11个问题）；护城河依赖数据积累但数据当前未启用过滤
心理健康类特有风险：
  • 过度依赖风险：青少年可能把AI陪伴替代真实人际关系——需要在对话设计中主动引导现实社交
  • 情绪误导风险：AI回复措辞不当可能强化错误认知或触发情绪——话术库需专业心理咨询师审核
  • 隐私泄露风险：情绪数据是高度敏感的个人信息，家长/老师端的可见权限边界必须清晰
  • 危机漏报风险：关键词检测有漏洞（绕过/变体表达），不能完全依赖AI判断，需要人工复核机制

Q10 减压星球的战略定位是什么？护城河在哪里？ ▶
2026.06.11
战略定位：青少年情绪健康的「情感基础设施」，不是工具，是关系。

市面上竞品（好奇岛、心晴、Soul学生版）都在做工具层——测评、内容、匿名社区。减压星球的差异化方向是把工具嵌入一套有叙事的世界观，让学生和IP建立情感连接，而不只是使用功能。
护城河一：时间积累的情绪档案每一条情绪打卡、每一次对话都在积累专属于这个学生的档案。初升高时罗素能引用吉塔陪伴初中时的对话记录——这种跨年级的连续性，是任何新产品无法在短时间内复制的。
护城河二：B2B学校关系学校采购绑定了场景——上课前签到、心理老师用后台——用户不是个人消费者而是学校，迁移成本极高。
护城河三：科大讯飞语音技术壁垒语音对话是青少年用户的首选交互方式，讯飞在语音识别和TTS上的技术储备是竞品短期内很难追上的。

Q1 年段适配是三套前后端，还是一套系统？ ▶
2026.06.11
A：一套系统，一套代码，按 phaseCode 渲染不同视图层。
更接近同一个网站做了三个皮肤，而不是三个不同产品。

APK 里已有 phaseCode（小学/初中/高中）和 gradeCode（具体年级），登录时已知，只是现在没有用它来过滤内容。
后端：一次「内容打标」所有内容、任务、测评模板打上适用年级标签；API返回内容时带上 phaseCode 过滤参数。这不是三套API，是一套API加了一个筛选维度。
前端：三套「视图配置」首页布局配置按 phaseCode 读取不同配置文件；组件本身复用，只是排列密度、文字描述、功能入口的可见性不同。
技术实现四步：登录时读 phaseCode → 读视图配置JSON → 组件加density prop → 后端内容打标过滤。改动范围：写三份JSON + 给内容打标签，不需要重写业务逻辑。

最值得关注的点：年级衔接。初升高时罗素能引用吉塔对话里的兴趣记录——这是护城河靠时间积累的核心逻辑。

补 私密情绪日记和危机干预系统有没有冲突？学生日记会被老师看到吗？ ▶
2026.06.11
A：日记默认不进入危机检测，边界非常清晰。

E2提案三明确规定：私密日记纯文字，无公开选项，无老师/家长可见权限——这是用户信任的底线。

唯一例外（学生主动授权）：学生可以主动勾选「让吉塔看看这篇」，此时吉塔会在下次对话中提及，但不会通知老师/家长。

危机干预只在AI对话中触发（非日记）：在对话里出现危机关键词才触发三级协议。日记和对话是两个独立的数据空间。

依据：《个人信息保护法》第十三条·知情同意原则。用户协议需明确告知「对话内容可能触发支持流程，日记内容不会」。
