Preface · 设计前提
澄清一个认知误区
📐 不是「三套前后端」，是「一套系统，按 phaseCode 渲染不同视图层」
很多人看到「三套年级视图」，第一反应是：要不要给小学生、初中生、高中生各开发一套 App？
答案是不需要。这更像同一个网站换了三个皮肤——组件复用，只改排列、密度和可见性。

APK 里已经有 phaseCode（小学 / 初中 / 高中） 和 gradeCode（具体年级），用户登录时系统已知这两个值，只是现在没有用它们来过滤内容和调整视图。这是一个「快赢」级别的升级——基础数据已在，缺的是「把它用起来」的设计决策。

Phase Design · 三阶段体验设计
每个阶段在意什么
心理发展阶段决定了产品应该说什么语言——不是功能多少的问题，是「和谁说话」的问题
🌱
小学版
phaseCode = primary · 小学3-6年级
「规则期：世界是安全的、公平的、有规律的——游戏是最好的语言。」
首页：大图形情绪签到（emoji 脸谱代替文字），脑之岛地图作为核心视觉主体
AI对话：小星主导，短问短答，不超过2-3句交换，图片+文字混合
脑之岛：游戏感最强，迷雾解锁明显，每完成1个关卡立即给奖励反馈
内容：心理知识以绘本/动画为主，不出现「压力」「焦虑」等词汇
隐藏：生涯规划、志愿填报、咨询预约、所有文字密集型内容不显示
家长联动最强：家长端实时看到孩子情绪签到，共同完成「亲子任务」
🌀
初中版
phaseCode = middle · 初中1-3年级
「身份认同危机期：「我是谁」比「我要做什么」更重要——被理解比被指导更重要。」
首页：情绪签到仍显眼，但加入「今日心情故事」文字入口；AI对话入口更大
AI对话：吉塔权重提升，对话深度加深，支持多轮长对话，支持「匿名倾诉模式」
探索Tab：人际关系、青春期困惑类内容显示；「朋友间的矛盾」「被孤立」等主题可见
朋友圈：功能开放，情绪动态可分享给班级群（学生自主选择可见范围）
生涯：仅显示「探索兴趣」轻量版，不显示选科/志愿填报模块
家长联动：汇报粒度变粗（「整体情绪趋势」而非每日签到），保护青春期隐私
🎯
高中版
phaseCode = high · 高中1-3年级
「压力峰值期：他们知道选择有代价——需要数据、逻辑和被人相信能做出好决定。」
首页：信息密度最高，出现「距高考 N 天」倒计时（可关闭），数据卡片可见
生涯模块全开：选科矩阵、职业地图、志愿模拟、生涯档案——罗素核心场景
AI对话：三IP全部显示，可手动选择；罗素对话支持导出「分析报告」
心理测评：加入「学业压力专项评估」，报告更详细，可与咨询老师共享
家长联动：生涯报告可主动分享给家长（学生授权），志愿方案可导出PDF
游戏化降权：脑之岛保留但不做首屏主推，不用「打怪升级」语气说学业

心理测评 趣味版（「你是什么动物」形式），无临床量表 标准情绪量表 + 人际关系量表，报告对学生呈现 增加学业压力专项评估，报告可共享给咨询老师
朋友圈 仅显示成就勋章，不支持情绪内容分享 情绪动态可分享，可见范围：仅班级 / 仅自己 同初中版，额外可分享「生涯成就」（如完成选科报告）
家长端联动 关键差异 最详细：每日签到 + 脑之岛进度 + 亲子任务 周报形式：情绪趋势折线 + 使用时长，不透传具体对话 学生主动授权后可见：生涯报告 / 测评报告（非日常）
游戏化语言 全面游戏化：「冒险者」「解锁宝藏」「升级」 部分游戏化：勋章、连击，但去掉「冒险者」称谓 最低游戏化：保留勋章，去掉升级称号，不用「游戏」框架说学业
咨询预约 不显示（无相关需求） 显示（但标注「部分学校开通」），危机时由吉塔引导 显示，加入「高考前特别咨询」快捷入口

Technical Design · 技术实现方案
一套代码如何支撑三个视图
改动范围小、风险低——核心是「读配置」，而不是「重写逻辑」
前端实现逻辑
Step 1
登录时
用户登录后，服务端返回 phaseCode + gradeCode（已有字段，不需新增）。
前端存入全局状态（Vuex store / localStorage），整个 session 内有效。
Step 2
读配置
首页、探索Tab、底部导航各维护一份 视图配置文件（JSON），结构如下：
{ "primary": { "showCareer": false, "showBrainIsland": true, "heroModule": "emotion", ... }, "middle": {...}, "high": {...} }
前端启动时根据 phaseCode 读取对应配置，决定模块的 显隐 / 排列顺序 / 文案变体。
Step 3
组件层
所有组件代码不改动。组件接收 props 控制样式密度（density: "low" | "mid" | "high"），
内部根据 density 决定是否显示文字标签、数据图表、多级交互。
例如「情绪签到」组件：density=low 只显示 emoji；density=high 显示 emoji + 文字 + 趋势入口。
Step 4
内容层
后端 CMS 给所有内容条目加一个字段：targetPhase: ["primary"] | ["middle"] | ["high"] | ["middle","high"]。
API 在返回内容列表时自动按用户的 phaseCode 过滤，不需要前端做任何筛选逻辑。
一次打标工作，后续所有新内容录入时同步维护。
需要新增 / 改动的工作
✦ 写三份视图配置 JSON（一次性工作）
✦ 给现有内容打年级标签（运营工作）
✦ 组件加 density prop 支持（前端改动）
✦ API 内容列表加 phaseCode 过滤（后端改动）
✦ 年级升级时的数据迁移逻辑（见下节）
不需要做的事
✗ 开发三套独立 App 或独立代码库
✗ 重写现有组件逻辑
✗ 新增三套 API 端点
✗ 三套独立的用户账号体系
✗ 三套独立的 UI 设计稿

技术方案基于 APK 已有 Vue.js WebView 架构推演；具体实现需结合现有前端状态管理方案确认。

Grade Transition · 年级衔接设计
从小学升初中、从初中升高中
phaseCode 更新时，不只是换一套视图——学生积累的数据和记忆要平滑过渡，不能「消失」
🌱→🌀
小学升初中
phaseCode: primary → middle · 通常发生在暑假开学后账号更新年级时
数据保留：脑之岛解锁记录、勋章墙、情绪签到历史全部保留，在「成长回顾」中可查。
视图切换：系统检测到 phaseCode 更新后，下次启动时触发「成长」动画+引导页：「你升初中了！我们一起认识一下新的伙伴……」吉塔首次自我介绍。
内容迁移：小学专属内容进入「档案」，不再显示在探索首屏；初中新内容自动上线。
家长联动变化：系统提示家长端「孩子已进入初中版本，数据汇报粒度调整为周报」，并说明原因（保护青春期隐私）。
🌀→🎯
初中升高中 · 最重要的一次衔接
phaseCode: middle → high · 生涯模块首次激活，数据积累开始计时
数据保留：所有情绪档案、吉塔对话记录、测评报告全部保留，进入「成长档案」。这是高中阶段生涯报告的历史输入。
视图切换：首次打开高中版，罗素进行自我介绍：「我是罗素，从今天开始，我会帮你在接下来三年里，把所有和升学相关的问题理清楚——先从认识你自己开始。」直接引导完成第一次生涯兴趣测评。
生涯档案初始化：系统基于初中阶段的兴趣探索记录（如有），预填生涯档案的兴趣方向初始值，罗素提示「这是基于你初中时和吉塔聊到的内容，你觉得准确吗？」
吉塔角色延续：吉塔不消失——高考压力场景下，吉塔的权重比初中期更重要。两个 IP 并存，罗素负责决策，吉塔负责情绪。
为什么年级衔接是「情感基础设施」的核心时刻
一个用AI星球三年的初中生，升入高中时，罗素能说「上次你在吉塔那里提到你对写作有点兴趣，这在新闻、出版、内容策划方向都有对应——要一起看看吗？」

这句话的前提是：数据没有断，记忆没有丢。这是竞品复制不了的东西——不是因为技术复杂，而是因为它需要用户已经在这个产品里待了够长的时间。年级衔接设计得越平滑，这段时间就越长，护城河就越深。

年级更新触发机制：依赖学校在 B 端后台更新学生年级信息，或学生自行在设置中更新；建议每学年开学时系统发送「更新年级」提醒。phaseCode / gradeCode 的实际更新流程需结合学校 B 端管理权限设计。
