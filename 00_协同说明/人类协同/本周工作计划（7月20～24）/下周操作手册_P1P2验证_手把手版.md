# 下周操作手册：P1+P2 验证闭环（手把手版）

> **目标**：7.31前输出P1和P2的改进设计方案，证明能从1分提升到4分  
> **工具**：Claude Code（你现在用的这个）+ 浏览器  
> **时长**：每个痛点预计半天到一天

---

## 🎯 先理解一下要做什么

**现状**：
- P1首屏双卡片无主角，用户不知道点哪个 → 1分
- P2 AI对话切Tab就找不到 → 1分

**目标**：
- P1改成单主CTA设计（参考Headspace） → 4分
- P2改成全局悬浮入口（参考Replika） → 4分

**怎么验证达到4分？**
用三维度估分表里的标准，对着竞品参考分和Demo目标描述，看改完的设计是否达标。

## 📅 周一（7.28）：搞定 P1 首屏视觉引导

### 上午：让Claude生成P1改进方案

**第1步：打开Claude Code，新开对话**

**第2步：输入以下内容**（直接复制粘贴）：

```
我现在要改进减压星球APP的首屏设计。

【当前问题 P1】
打开App后看到两个平级的卡片（情绪打卡 + AI对话），用户不知道该点哪个，没有主次，评分1分。

【目标】
参考 Headspace 的意图驱动导航设计，做成单主CTA的首屏，让用户一眼知道主行动是什么，目标4分。

【竞品参考】
Headspace首屏：顶部一句话问"今天感觉怎么样？"，下方一个大按钮"开始冥想"，次级功能（历史记录、设置等）收到底部或侧边。

【技术约束】
PAD端，尺寸1920×1080，HTML/CSS实现，不改后端逻辑。

请生成一个可运行的HTML文件，展示改进后的首屏设计。
```

**第3步：等Claude生成HTML文件**

Claude会给你一个HTML文件，里面是改进后的首屏设计。

**第4步：浏览器打开HTML，截图**

打开生成的HTML文件，看看效果，截图保存（命名：P1改进方案_v1.png）

---

### 下午：评估方案 + 迭代改进

**第5步：对照评分标准自己打分**

打开 `三维度体验基线估分表.html`，找到P1那一行，看看：
- 竞品参考分4分的描述是什么？（"Headspace单主CTA"）
- Demo目标分4分的要求是什么？

看看Claude生成的方案：
- 是否有明确的单主CTA？
- 是否让用户一眼知道该做什么？
- 次级功能是否收起来了？

如果觉得还不够4分，就继续下一步。

**第6步：告诉Claude哪里不够好**

在Claude Code的对话里继续说：

```
我看了一下，有几个问题：
1. [具体问题，比如：主按钮不够突出]
2. [具体问题，比如：顶部文案太长]

请改进一下。
```

**第7步：重复第3-6步，直到满意**

每次改进都打开HTML看效果，直到你觉得达到4分标准为止。

**第8步：保存最终方案**

把最终版的HTML文件重命名为：`P1首屏改进_最终版_0728.html`  
把所有版本的截图放到一个文件夹：`P1改进过程_0728/`

重复周一的流程，但换成P2的内容：

请生成一个可运行的HTML文件，展示改进后的设计。文件里要包含3-4个不同的页面（首页、个人中心、训练页等），证明AI入口在每个页面都能被看到和点击。

然后重复周一的第3-8步，输出最终方案。

## ❓ 常见问题

**Q1：Claude生成的HTML我不满意怎么办？**  
A：直接在对话里说"这里不好，改成XXX"，Claude会改。你可以无限次迭代，直到满意为止。

**Q2：我怎么知道达到4分了？**  
A：对照竞品参考分的描述，如果你生成的方案和竞品水平差不多，就算达标。

**Q3：HTML文件我能不能自己改？**  
A：可以，但不建议。因为你改完之后如果还想让Claude继续改，它看到的是旧版本，会乱。建议全程让Claude改。

**Q4：这个方案最后要不要交给开发？**  
A：不用。这周的目标是"验证闭环能不能跑通"，输出的是设计方案验证，不是开发交付。真正要开发的时候，你再基于这个验证结果重新整理交付稿。

**Q5：周五汇报要讲多久？**  
A：5-10分钟。重点讲：改了什么、为什么这么改、改完能达到什么水平、有没有遇到什么问题。
